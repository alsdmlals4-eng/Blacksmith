## 재현 방법

```text
python tools/simulate_enhancement_balance.py --trials 1000 --seed 29029 --output <임시 결과 JSON 경로>
```

- 입력: 강화·이정표·재료·수식어·제작 JSON 다섯 개의 SHA-256과 Git 커밋을 결과 JSON에 기록한다.
- 난수: Python Mersenne Twister, trial별 `seed + index`.
- 동등성: 난수 알고리즘 자체가 아닌 고정 roll 반례로 `EnhancementSession`의 유지·실패 보정·하락·파괴·폭주 차단·특수 비용·판매가 경로를 검증한다.
- 시나리오: 철검 공격력 20, 숫돌, 촉매 없음, 특수 강화 완벽 정밀 보정 `+0.20`, 자원 제한 없음, 단일 무기 파괴 시 종료.

이 기준선은 자동 반복·재료 부족 fallback·제작 품질/피버 조합·실제 손맛을 대표하지 않는다. 대용량 원본 JSON은 소스 저장소에 넣지 않고 실행 산출물로 보관한다.

## 해석과 다음 결정

- `KEEP`: +10~+30은 현행 단일 무기 기준에서 선택 차이가 읽히며, 즉시 수치 변경 근거가 부족하다.
- `TUNE 후보`: +70 이후 균형·폭주의 단일 무기 완주 가능성이 매우 낮고, +100 안정 단조의 비용 분포가 급격히 상승한다.
- `TEST_IN_PLAY`: 파괴·재시작의 손실 피로, 자동 단조 반복의 실제 목표 도달 시간, 제작 품질·피버의 보상 체감은 이 결과만으로 판정할 수 없다.
- `REJECT`: 아직 없음. 후보 수치안은 만들지 않았고, 다음 실험에서 성공률·비용·하락·파괴 중 한 변수군만 바꿔 동일 시드와 시나리오로 비교한다.
