### 시장 포지션

- 가장 가까운 표면 경쟁작은 `Blacksmith Master`다. 그러나 해당 게임은 작업장·생산 체계 관리에 중심이 있다.
- 가장 가까운 구조적 참고점은 `Potion Craft`의 직접 제작·고객 적합도, `Holy Potatoes!`의 영웅 판매·재방문, `Anvil Saga`의 평판·세계 결과다.
- 모바일에서는 `Forge Ahead`와 `Smithing Master`가 즉시 입력과 대량 성장 기대를 형성하지만, Blacksmith는 단순 탭·방치·전투 RPG가 아니라 `장비 한 점의 역사`에 집중해야 한다.

### 권장 차별화 문장

> 강화 수치를 올리는 게임이 아니라, 내가 만든 장비가 다른 사람의 손에서 만든 역사가 다시 나의 명성이 되는 게임.

### 차별화의 실제 방어선

대장간 소재, 강화 확률, 수식어 후보는 모방하기 쉽다. 상대적으로 방어 가능한 지점은 다음의 연결 구조다.

`제작 선택 → 영구 출생 품질 → 강화 경로 → 소유자 → 세계 사건 → 장비 이력 → 제작자 명성 → 다음 의뢰`

따라서 VRIO의 모방 난도는 단일 기능이 아니라 데이터 계약·사건 문법·콘텐츠 제작 파이프라인에서 만들어야 한다.

## 6. 적대적 검토 루프

### Loop 1 - 정체성 최소성·범위 공격

**실패 가정**

`장비 생애`라는 표현 아래 +100, 3계열, 세계 시뮬레이션, 이벤트, 관전, 피로도, 숙련도와 자동 단조가 모두 코어로 굳어져 제작 범위가 폭발한다.

**발견**

- 정확한 +100은 정체성보다 콘텐츠·밸런스 목표에 가깝다.
- 판매 장비 전부를 계속 시뮬레이션하면 영구 이력보다 관리 시스템이 중심이 된다.
- 자동 단조가 고위험 구간까지 수행하면 강화 선택을 제거한다.
- 3계열 장기 목록이 확정 수량처럼 읽힐 수 있다.

**비판 검증**

- `+10` 성장 리듬과 점증 위험은 정체성에 필요하나 숫자 100은 대체 가능하므로 지적 수용.
- 세계 잔존을 삭제하자는 비판은 프로젝트 약속을 훼손하므로 기각. 대신 영구 기록과 활동 상태를 분리.
- 자동 단조 전체 삭제는 반복 피로를 키우므로 기각. 위험·이정표 강제 정지로 제한.
- 무기·방어구·악세서리 방향은 승인됐으므로 유지하되 철검 외 수량은 단계별 검증으로 이동.

**반영**

- +100을 `현재 제품 목표`로 이동.
- `ACTIVE / EVENT_ELIGIBLE / DORMANT / HISTORICAL` 생애 상태 추가.
- 자동 단조 강제 정지 가드레일 추가.
- 장비 종류 수량 선확정 금지.

**회귀 재검토**

단일 대장장이, 직접 제작, 영구 완성도, 1회 강화 판단, +10 수식어, 세계 기록, 명성 환류와 피로도는 유지됐다. 코어 정체성 손실 없음.

### Loop 2 - 플레이어 행동·PoC 공격

**실패 가정**

PoC 참가자는 철검을 기억하지 못하고, +10까지 생각 없이 누르며, 낮은 완성도는 버리고, 결과 보고에서는 골드만 확인한다.

**발견**

- 기존 PoC는 결과가 즉시 도착해 장기 기대를 검증하지 못했다.
- +10까지 파괴가 없고 최종 목표가 +10이면 멈춤 이유가 약했다.
- 결과 계산의 원인 설명 계약이 없었다.
- 숙련도 성장까지 포함해 품질·정밀 마감 효과를 분리하기 어려웠다.
- 낮은 완성도 폐기 행동을 실패 조건으로만 적고 측정 방법이 없었다.

**비판 검증**

- 지연 결과·재방문은 핵심 약속을 직접 검증하므로 수용.
- PoC에 파괴를 넣자는 제안은 첫 가설을 흐리므로 기각. +5 충족선과 +10 선택의 기회비용으로 대체.
- 숙련도 시스템 삭제는 승인 방향을 훼손하므로 기각. 첫 PoC에서 값만 고정.
- 완성도 재선택 허용은 영구 출생 계약을 훼손하므로 기각. 낮은 등급 지속 플레이의 가치를 검증.

**반영**

- 결과를 최소 하루 지연하고 같은 검투사 재방문 포함.
- 의뢰 기본 충족 +5, 선택 목표 +10.
- 결과 기여도 설명 추가.
- 고정 숙련도 프로필 사용.
- 낮은 등급 폐기율·장비 회상·멈춤 이유·결과 확인 행동을 정량 게이트로 추가.

**회귀 재검토**

PoC는 철검 1종·검투사 1명·3일 이내로 유지돼 범위가 증가하지 않았다. 전쟁·관전·베팅·복원·다수 고객은 계속 제외된다.

## 8. KEEP / AMPLIFY / CHANGE / DEFER

### KEEP

- 한 명의 대장장이
- 빠른 제작 입력과 묵직해지는 강화 연출
- 버튼 한 번당 강화 결과 한 번
- 영구 완성도
- +10 수식어 성장
- 판매 장비의 영구 이력과 명성 환류
- 피로도 중심 날짜 진행과 50% 이월
- 직원·직접 전투·작업 예약 제외

### AMPLIFY

- +5에서 멈출지 +10을 노릴지 선택하는 순간
- 수식어가 고객·사건 결과에 미치는 설명 가능한 영향
- 낮은 완성도 장비의 역사적 반전
- 같은 고객의 재방문과 이전 장비 언급
- 강화 단계에 따른 음향·정적·공명 변화
- 장비 카드보다 장비 이력·소유자·사건 회고

### CHANGE

- 정확한 +100을 프로젝트 코어 불변으로 취급하는 문장
- 모든 판매 장비를 계속 적극 시뮬레이션하는 해석
- PoC의 즉시 결과 보고
- 숙련도 성장·완성도·정밀 마감을 한 실험에서 동시에 검증하는 구조
- 자동 단조가 특수·위험 강화 판단을 대체하는 구조
- 결과 화면에서 골드·공격력 숫자가 원인 설명보다 앞서는 위계

### DEFER

- 방어구·악세서리 실제 구현
- +100 최종 상한 확정
- 자동 단조 재설계
- 다수 고객·세력·시장
- 전쟁·전체 관전·베팅
- 대표작 복원
- 생활·산업 도구와 소모품
- 장비 수량과 사건 수량의 대규모 확장
