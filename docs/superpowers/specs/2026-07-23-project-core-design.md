## 1. 최종 프로젝트 정체성

### 1.1 정체성 한 문장

> 한 명의 대장장이가 제한된 하루 작업량으로 장비 한 점을 직접 만들고, 매 강화마다 멈출지 더 도전할지 선택하며, `+10` 이정표에서 장비의 성질을 벼린 뒤, 그 장비가 다른 이의 손에서 쌓은 역사가 다시 명성과 다음 의뢰로 돌아오는 모바일 제작 게임.

### 1.2 뾰족한 재미

> 내가 만든 장비의 운명을 직접 결정하고, 세상에서 그 선택의 결과를 되돌려 받는 재미.

### 1.3 시장 차별화 문장

> 장비를 대량 생산하는 게임이 아니라, 장비의 출생·성장·소유·사건 기록을 제작하는 대장장이 게임.

### 1.4 핵심 플레이어 약속

- 내가 직접 만든 장비는 판매 순간 사라지는 소모품이 아니다.
- 장비의 완성도, 강화 선택, 수식어와 세계 성과는 하나의 연속된 이력으로 남는다.
- 높은 수치만이 아니라 서로 다른 역사를 가진 장비를 다시 만들 이유가 생긴다.
- 플레이어는 직원 조직이 아니라 한 명의 대장장이만 운용한다.

## 2. 기획의 필연성

### Why - 전달할 경험

평범한 재료가 플레이어의 기술, 선택, 욕심과 위험을 거쳐 고유한 작품이 되고, 그 작품이 세상에서 인정받아 제작자의 정체성과 다음 기회를 만드는 경험을 전달한다.

### How - 경험을 만드는 전략

1. 제작 순간 영구 완성도 등급을 부여해 장비의 출생을 기록한다.
2. 일반 강화는 버튼 한 번당 결과 한 번으로 처리해 매번 멈춤과 추가 도전을 판단하게 한다.
3. `+10` 단위 특수 강화에서 보조재료·정밀 판정·후보 선택으로 수식어를 추가·성장·전문화한다.
4. 판매·납품·후원된 장비의 기록을 세계에 남기고, 일부 장비가 일반 활동과 토너먼트·전쟁 사건에 다시 등장하게 한다.
5. 세계 결과의 원인을 장비 적합도·완성도·강화·수식어·소유자·사건 맥락으로 설명한다.
6. 장비 성과를 골드뿐 아니라 명성, 관계, 희귀 재료, 재의뢰와 대표작 기록으로 환류한다.
7. 피로도와 날짜 진행으로 한 명의 대장장이가 오늘 무엇에 집중할지 선택하게 한다.

### What - 게임에 나타나는 결과

- 빠른 모바일 터치 제작과 강화 입력
- 강화가 높아질수록 묵직해지는 음향·정적·공명·결과 대비
- 무기·방어구·악세서리의 공통 제작·강화 문법
- 최초 제작 시 영구 완성도 등급
- `+10` 단위 일반 수식어 성장과 전문화
- 세계 잔존 기록, 소유자, 일반 활동, 토너먼트·전쟁
- 대장장이 통합 숙련도와 장비 계열 숙련도
- 피로도 중심 날짜 진행과 잔여 피로도 50% 이월

## 3. 코어 루프

### 3.1 장비 한 점의 생애

`목표·의뢰 확인 → 장비 종류·재료 선택 → 직접 제작 → 영구 완성도 결정 → 일반 강화 1회 판단 반복 → +10 특수 강화·수식어 선택 → 현재 완성 또는 추가 도전 → 판매·납품·후원 → 지연된 세계 활동·사건 → 장비 이력·명성·관계·보상 환류 → 더 중요한 작품 제작`

### 3.2 일일 운영

`새 날짜·세계 소식 확인 → 오늘의 우선순위 선택 → 제작·강화·복원 → 피로도 판단 → 판매·납품·후원 → 직접 하루 마치기 → 잔여 피로도 50% 이월 → 세계 활동·일정 진행`

### 3.3 행동·보상·진척

| 구분 | 내용 |
|---|---|
| 핵심 행동 | 제작 터치, 정밀 마감, 강화 1회 입력, 재료·촉매 선택, 수식어 후보 선택, 멈춤·추가 도전, 판매·납품·후원 |
| 즉시 보상 | 완성도, 강화 상승, 수식어, 외형·음향 변화, 가치 상승, 고객 반응 |
| 지연 보상 | 소유자 활동, 토너먼트·전쟁 성과, 장비 별칭, 명성, 관계, 희귀 재료, 상위 의뢰 |
| 메타 진척 | 대장장이 레벨, 장비 계열 숙련도, 설비, 제작법, 고객층, 대표작, 세계 기록 |

## 4. 보호 경계

### 4.1 `INVARIANT`

다음 요소는 제거하거나 본질적으로 바꾸면 같은 프로젝트로 보기 어렵다.

1. 플레이어는 한 명의 대장장이만 운용한다.
2. 플레이어가 장비를 직접 제작한다.
3. 장비는 최초 제작 시 영구 완성도 등급을 얻는다.
4. 일반 강화 버튼 한 번이 결과 한 번과 멈춤·추가 도전 판단을 만든다.
5. `+10` 단위에서 일반 수식어가 추가·성장·전문화된다.
6. 강화가 높아질수록 가치·위험·연출의 무게가 증가한다.
7. 판매된 장비의 출생·성장·소유·사건 이력이 세계 기록으로 남는다.
8. 일부 장비의 세계 성과가 대장장이 명성과 다음 제작 기회로 돌아온다.
9. 피로도가 하루 작업량을 제한하고 날짜 진행이 세계 사건을 움직인다.
10. 조작은 빠르지만 작품과 결과는 묵직하게 느껴져야 한다.

### 4.2 `CHANGEABLE`

- 현재 제품 목표인 최대 `+100`의 최종 상한과 구간 수
- 정확한 강화 확률·비용·피로도 수치
- 완성도별 확률·성능·가치 배율
- 보관함 크기, 개인 대표작 수, 세계 활동 장비 수
- 세부 장비 종류와 출시 순서
- 수식어 이름·효과·후보 수·희귀 후보 조건
- 고객·세력·토너먼트·전쟁의 수와 빈도
- 관전 길이, 베팅 한도와 제공 지역
- 결과 보고서의 세부 화면 표현

`+100`은 삭제된 것이 아니라 현재 구현·장기 제품 목표로 유지한다. 다만 프로젝트 정체성의 최소 불변 조건은 정확한 숫자 `100`이 아니라 `+10` 이정표의 반복, 점증하는 위험과 작품 성장이다. 최종 상한은 `+30` 위험 강화 PoC와 장기 반복 피로 검증 뒤 확정한다.

### 4.3 `REQUIRES_REAPPROVAL`

- 직원 또는 복수 대장장이 운용 추가

- 피로도·날짜 시스템 제거
- 베팅을 핵심 성장·수익 수단으로 전환
- 자동 단조가 고위험 판단과 특수 강화 선택을 완전히 대체하도록 변경

### 4.4 `OUT_OF_SCOPE`

- 플레이어 직접 전투

## 5. 플레이어 역할과 피로도

### 5.1 단일 대장장이

- 플레이어는 대장장이 한 명만 운용한다.
- 대장간 성장은 직원 수가 아니라 설비, 제작법, 재료 대응력, 숙련도와 고객층의 확장으로 표현한다.
- 자동 단조는 직원 생산이 아니라 이미 익힌 반복 입력을 줄이는 설비 편의 기능이다.

### 5.2 피로도 계약

- 제작·강화·중요 장비 복원은 피로도를 소비한다.
- 터치 횟수와 광클은 추가 피로도를 소비하지 않는다.
- 피로도 부족은 품질·강화 확률을 낮추지 않고 작업 시작만 제한한다.
- 날짜는 플레이어가 `하루 마치기`를 선택할 때만 진행된다.
- 작업 예약 기능은 사용하지 않는다.
- 남은 피로도의 50%를 다음 날 기본 피로도에 더한다.
- 별도 이월 상한은 두지 않는다. 작업하지 않고 계속 날짜를 넘길 경우 시작 피로도는 기본 최대치의 2배에 수렴한다.
- 날짜를 넘기면 의뢰·시장·토너먼트·전쟁 일정과 세계 활동도 함께 진행된다.
- 피로도 비축이 유일한 최적해가 되는지는 PoC에서 `무작업 날짜 비율`과 `이벤트 직전 몰아 제작` 행동으로 검증한다.

## 6. 제작 대상과 숙련도

### 6.1 장비 계열

1. 무기
2. 방어구
3. 악세서리

모든 계열은 공통 제작·완성도·강화 문법을 사용하고, 성능·수식어·세계 평가만 계열별로 달라진다.

### 6.2 장기 후보와 검증 순서

- 무기 후보: 검, 도끼, 창, 둔기, 활, 지팡이, 단검
- 방어구 후보: 흉갑, 투구, 장갑, 장화, 방패
- 악세서리 후보: 반지, 목걸이, 부적, 브로치, 허리띠, 팔찌
- PoC 1차: 철검만 사용
- 공통 구조 전이 검증: 흉갑과 반지
- 나머지 종류는 세 계열의 전이 검증이 통과되기 전 수량을 확정하지 않는다.

### 6.3 혼합형 숙련도

- 통합 대장장이 레벨: 전체 제작 확률, 재료 난이도 대응, 설비·제작법 해금
- 장비 계열 숙련도: 무기·방어구·악세서리별 완성도 분포와 희귀 후보 보정
- 세부 종류: 별도 경험치 막대 대신 제작법 해금과 제작 기록으로 관리
- 첫 PoC에서는 숙련도 성장을 검증 변수가 되지 않게 고정된 숙련도 프로필을 사용한다. 성장 체감은 별도 실험으로 분리한다.

## 7. 완성도 등급

### 7.1 역할과 등급

완성도는 장비가 처음 완성될 때 붙는 영구 품질 접두어이며 장비가 어떻게 태어났는지를 기록한다.

`엉성한 → 평범한 → 정교한 → 뛰어난 → 완벽한`

### 7.2 판정 요소

`통합 대장장이 숙련도 + 장비 계열 숙련도 + 정밀 마감 결과 + 재료 가공 난이도 + 제한된 무작위`

### 7.3 가드레일

- 최초 제작 뒤 재연마·재단조로 완성도를 다시 판정하지 않는다.
- 낮은 완성도도 모든 강화 이정표, 최고 수식어와 이벤트 참여에 접근할 수 있다.
- 완성도는 기본 전투 수치보다 판매 가치, 명성, 고객 평가, 외형·음향과 역사적 서술에 더 크게 작용한다.
- 낮은 완성도 장비도 세계 성과를 통해 대표작이 될 수 있다.
- 제작 실패 뒤 즉시 재제작만 반복하는 행동을 막기 위해 재료와 피로도는 정상 소비되며, PoC에서 낮은 등급 즉시 폐기율을 측정한다.

## 8. 강화와 수식어

### 8.1 공통 강화 골격

- 현재 제품 목표는 최대 `+100`이다.
- 전 계열은 동일한 성공·유지·하락·파괴 규칙을 사용한다.
- `+1~+10`: 학습 구간, 실패 시 기본적으로 단계 유지
- `+11` 이상: 단계 하락 가능
- `+30` 이상: 장비 파괴 가능
- 실패 보정은 누적되고 성공 시 초기화된다.
- 균형·안정·폭주 단조는 전 계열이 동일한 계약을 사용한다.

### 8.2 일반 강화

`강화 버튼 입력 → 비용·피로도 확인 → 1회 판정 → 결과와 다음 가치 표시 → 다시 누르거나 멈춤`

- 버튼 한 번당 한 번의 결과를 낸다.
- 보조재료·촉매·정밀 판정은 사용하지 않는다.
- 초기 연출은 빠르고 가볍다.
- 강화가 높아질수록 저음, 정적, 공명, 충격과 결과 대비가 점차 묵직해진다.
- 강화 결과를 확인하기 전에 다음 강화가 자동 실행되지 않는다.

### 8.3 `+10` 특수 강화

`보조재료 선택 → 수식어 계열 결정 → 촉매 선택 → 효과·비용·위험 확인 → 정밀 강화 → 기본·희귀 후보 생성 → 최종 후보 선택`

- 기본 후보는 재료와 현재 성장 경로를 통해 예측 가능해야 한다.
- 특정 조합이나 완벽 판정에서 희귀·숨은 후보가 추가된다.
- 보조재료가 수식어 계열을 정한다.
- 정밀 판정이 후보 수·등급·희귀 변형 확률을 바꾼다.
- 플레이어가 최종 후보를 선택한다.
- 핵심 선택의 효과를 확률이 완전히 지우지 않도록 결과 영향의 최소치를 보장한다.

### 8.4 수식어 이정표

현재 장기 목표:

- `+10`: 첫 수식어 1티어
- `+20`: 첫 수식어 2티어·전문화
- `+30`: 두 번째 수식어 1티어
- `+40`: 두 번째 수식어 2티어·전문화
- `+50`: 세 번째 수식어 1티어
- `+60`: 세 번째 수식어 2티어·전문화
- `+70~+90`: 각 수식어 3티어
- `+100`: 세 수식어의 4티어와 전설 특성 완성

### 8.5 자동 단조 가드레일

자동 단조는 `CORE_SUPPORT`이며 다음을 우회하지 않는다.

- 실제 시도마다 골드·재료·피로도·확률을 동일하게 적용한다.
- 모든 `+10` 특수 강화 전에 멈춰 플레이어 선택을 요구한다.
- 하락·파괴 위험이 처음 열리는 구간과 플레이어가 지정한 가치 기준에서 멈춘다.
- 피로도·골드·재료 부족 시 중지하며 날짜를 자동 진행하지 않는다.
- 대표작 또는 개인 지정 장비에는 기본적으로 자동 고위험 강화를 적용하지 않는다.

## 9. 세계 잔존 장비와 결과 인과성

### 9.1 영구 기록과 활동 상태 분리

판매·납품·후원된 모든 장비는 출생·성장·소유·중요 사건의 영구 기록을 가진다. 다만 모든 장비를 계속 적극적으로 시뮬레이션하지 않는다.

장비 생애 상태:

1. `WORKSHOP`: 대장간 보유
2. `ACTIVE_OWNER`: 소유자가 현재 사용하며 일반 사건 후보
3. `EVENT_ELIGIBLE`: 토너먼트·전쟁 등 중요 사건 후보
4. `DORMANT`: 세계에 존재하지만 일반 사건 빈도를 낮춘 상태
5. `HISTORICAL`: 활동은 종료됐지만 이력은 영구 보존
6. `BROKEN_OR_LOST`: 파손·분실 기록과 복원 후보 상태

활동 장비 수는 최근 판매, 중요 고객, 개인·시스템 대표작, 예정 이벤트와의 관련성으로 제한한다. 휴면·역사 장비도 기록은 삭제하지 않으며 중요 사건에서 다시 활성화될 수 있다.

### 9.2 결과 계산 원칙

개념적 결과 구조:

`소유자 기본 역량 + 의뢰 적합도 + 장비 종류·수식어 적합도 + 강화·완성도 영향 + 사건 맥락 + 제한된 변동성`

- 핵심 정답은 장비 적합도와 플레이어 선택에서 나온다.
- 무작위는 승패의 세부 강도, 추가 보상, 부상·손상과 이변을 바꾸되 선택의 의미를 완전히 반전시키지 않는다.
- 결과 보고서는 최소한 `효과가 있었던 선택`, `부족했던 조건`, `발생한 사건`을 구분해 보여준다.
- 플레이어가 다음 제작에서 무엇을 바꿀지 설명할 수 있어야 한다.

### 9.3 일반 활동과 중요 이벤트

- 일반 활동: 게임 날짜 경과에 따라 짧은 보고로 처리
- 중요 이벤트: 토너먼트·전쟁처럼 일정이 사전 공개되는 사건
- 개입 방식: 핵심 NPC 직접 의뢰, 선택적 후원, 시장 수요 변화의 혼합형
- 한 이벤트의 주요 상호작용은 1~2개로 제한
- 결과 확인은 기본이며 관전과 게임 내 재화 베팅은 선택 기능
- 플레이어 직접 전투 조작은 사용하지 않는다.
- 베팅은 제작보다 효율적인 주 성장 수단이 될 수 없으며 출시 지역·플랫폼·연령 정책 검토 전 구현하지 않는다.

### 9.4 명성 환류

- 즉시 명성: 완성도, 강화, 의뢰 적합도, 수식어, 희귀성
- 사후 명성: 토너먼트 순위, 전쟁 공헌, 소유자의 생존·성공, 장기 사용과 역사적 사건
- 플레이어 통제 밖의 패배 때문에 기존 명성을 크게 차감하지 않는다.
- 실패는 재의뢰, 분석 정보, 수요 변화, 관계 변화와 복원 사건으로 전환한다.

## 10. 대표작과 복원

### 10.1 혼합형 대표작

- 시스템 대표작: 세계 성과로 자동 등록되는 역사적 장비
- 개인 대표작: 플레이어가 제한된 수만큼 직접 지정하는 애착 장비
- 대표작 지정 자체는 전투 성능·판매가·강화 확률을 직접 높이지 않는다.

### 10.2 선택형 복원

- 일반 장비: 파손 시 활동을 종료하고 역사 기록만 보존
- 중요 장비: 대표작, 고강화, 희귀 수식어, 유명 소유자 등 조건을 만족하면 복원 의뢰 후보
- 복원 시 완성도, 강화 단계, 수식어, 별칭과 세계 이력을 유지
- 복원에서 추가 확률 파괴를 사용하지 않는다.
- 복원 난도는 희귀 재료, 피로도, 정밀 작업과 선택 기회비용으로 표현한다.

## 11. PoC 확정안 - 장비 한 점의 생애

### 11.1 가장 위험한 가설

> 플레이어가 직접 만든 철검을 기억하고, 자신의 제작·강화·수식어 선택이 검투사의 결과에 영향을 줬다고 이해하며, 하루 이상 지연된 후속 결과와 재방문을 다시 확인하고 싶어 하는가?

### 11.2 최소 플레이 흐름

`검투사 의뢰 확인 → 철검 제작 → 영구 완성도 결정 → +5 권장선까지 일반 강화 → 현재 납품 또는 +10 도전 판단 → +10 특수 강화·수식어 선택 → 납품 → 하루 진행 → 지연된 경기 결과 보고 → 장비 이력·명성 갱신 → 같은 검투사의 재방문`

### 11.3 의도적인 멈춤 판단

- 의뢰는 `+5`에서 기본 충족되며 `+10`은 더 높은 보상과 명성을 노리는 선택이다.
- 일반 강화 실패는 단계 유지지만 골드·피로도와 이벤트 기한을 소비한다.
- 플레이어는 현재 적합도, 남은 피로도, 납품 기한과 +10 수식어 가능성을 보고 멈춤·추가 도전을 선택한다.
- PoC에서는 파괴보다 `기회비용과 결과 기대`로 첫 멈춤 판단을 검증한다.

### 11.4 포함 범위

- 철검 1종
- 검투사 1명과 재방문 1회
- 3일 이내의 최소 일정 구조
- 완성도 5등급
- 고정된 통합·무기 숙련도 프로필
- `+1~+10` 강화
- 화염·날카로움 수식어 2계열, 계열별 핵심 후보 2개
- 피로도, 수동 날짜 진행, 잔여 피로도 50% 이월
- 장비 납품, 하루 이상 지연된 결과 보고, 영구 장비 이력
- 골드, 명성, 관계, 재방문 의뢰
- 결과 기여도 설명

### 11.5 제외 범위

- 숙련도 성장 자체의 재미 검증
- 방어구·악세서리
- `+11` 이후 하락과 `+30` 이후 파괴
- 자동 단조
- 상인 납품
- 전쟁·관전·베팅
- 대표작·복원
- 다수 고객과 복잡한 시장
- 실시간 전투 시뮬레이션

### 11.6 테스트 방법

- 권장: 외부 신규 플레이어 6명 이상, 각자 독립 세션
- 6명 미만이면 정성 증거는 기록하되 판정은 `PARTIAL`
- 완성도 분포는 테스트 시드로 저·중·고 등급을 의도적으로 배분하되 플레이어에게 결과를 조작하지 않는다.
- 플레이 중 설명을 대신하지 않고 행동, 머뭇거림, 재확인, 발언과 선택 이유를 기록한다.
- 자기보고와 실제 행동을 분리한다.

### 11.7 필수 관찰 지표

- 장비 이름·완성도·수식어 중 하나 이상을 결과 뒤 기억하는가
- `+5`에서 멈추거나 `+10`을 시도한 이유를 자기 말로 설명하는가
- 검투사 결과에 영향을 준 자신의 선택을 하나 이상 지목하는가
- 낮은 완성도 장비를 즉시 버리고 재제작만 반복하는가
- 납품 직후와 하루 뒤 결과 보고를 자발적으로 확인하는가
- 피로도를 모으기 위해 무작업 날짜 넘기기를 반복하는가
- 재방문 의뢰에서 이전 장비의 역사와 새 제작 목표를 연결하는가

### 11.8 초기 통과 신호

6명 기준:

1. 5명 이상이 자신의 철검을 다른 철검과 구분해 기억한다.
2. 4명 이상이 멈춤·추가 도전 이유를 설명한다.
3. 4명 이상이 결과에 영향을 준 제작·수식어 선택을 설명한다.
4. 4명 이상이 지연된 결과 또는 같은 검투사의 재방문을 기대하거나 자발적으로 확인한다.
5. 낮은 완성도 세션 참가자 중 과반이 즉시 폐기만 하지 않고 강화·납품·보관 중 하나를 선택한다.
6. 무작업 날짜 넘기기를 반복적으로 최적 행동으로 선택한 참가자가 1명을 넘지 않는다.
7. 결과 보고에서 골드보다 장비 이력·소유자·사건 중 하나를 먼저 언급한 참가자가 4명 이상이다.

### 11.9 실패 조건과 판정

- 완성도가 색상·능력치 배율로만 느껴진다.
- 일반 강화가 생각 없이 반복하는 버튼 작업이 된다.
- `+5`와 `+10` 사이에 실제 판단이 없다.
- 수식어 후보의 용도 차이를 설명할 수 없다.
- 결과가 장비 선택과 무관한 랜덤 보상처럼 느껴진다.
- 납품 순간 장비에 대한 관심이 사라진다.
- 낮은 완성도 장비가 즉시 폐기된다.
- 피로도 때문에 날짜만 반복해서 넘긴다.
- 결과 보고보다 골드 숫자만 중요하게 느껴진다.

결과는 항목별로 `KEEP / AMPLIFY / CHANGE / REJECT / TEST_AGAIN`으로 판정한다.

## 13. SWOT·VRIO 현재 판정

### 13.1 SWOT

**강점**

- 제작·강화·세계 이력이 하나의 감정 루프로 연결된다.
- 전 장비 공통 규칙으로 학습 복잡도를 낮춘다.
- 빠른 모바일 입력과 장기 애착을 함께 노린다.
- 직원 없이 한 명의 대장장이 판타지에 집중한다.

**약점**

- 장비 이력과 세계 사건 콘텐츠 제작량이 커질 수 있다.
- 확률 강화와 피로도가 함께 있을 때 피로감이 중첩될 수 있다.
- 낮은 등급 장비도 의미 있게 만드는 밸런스가 어렵다.
- 장기 강화 곡선이 반복적으로 느껴질 위험이 있다.

**기회**

- 관리형 대장간 게임과 단순 탭·방치형 제작 게임 사이에서 `장비의 세계 생애`로 구분할 수 있다.
- 토너먼트·전쟁 일정이 장비 수요와 재방문 동기를 만든다.
- 장비 이력과 대표작은 공유·회고·수집 콘텐츠로 확장 가능하다.

**위기**

- 강화 중심 게임이나 방치형 경영 게임으로 오인될 수 있다.
- 관전·베팅·세계 시뮬레이션이 코어보다 커질 수 있다.
- 장비별 서사가 템플릿 반복으로 보일 수 있다.
- 확률·베팅 표현은 출시 지역과 플랫폼 정책 검토가 필요하다.

### 13.2 VRIO

| 항목 | 현재 판정 | 다음 증거 |
|---|---|---|
| Value | `PROMISING / UNVERIFIED` | PoC에서 제작 선택이 결과 기대와 재플레이 행동으로 연결되는지 |
| Rarity | `PARTIAL` | 경쟁작 대비 `영구 장비 이력·소유자·사건 환류`가 명확한 차별 메시지인지 |
| Imitability | `PARTIAL` | 단순 수치가 아니라 장비·소유자·사건·명성이 연결된 데이터·콘텐츠 문법을 구축할 수 있는지 |
| Organization | `NOT_PROVEN` | 소규모 제작 규모에서 사건 템플릿·저장·밸런스·QA 파이프라인을 유지할 수 있는지 |

현재 결론은 `CORE_CONFIRMED / PRODUCTION_NOT_GREENLIT`이다. 코어 방향은 확정하지만 외부 플레이 증거 없이 프로덕션 범위를 확대하지 않는다.
