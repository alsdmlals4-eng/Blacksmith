## 1. 프로젝트 정체성

### 1.1 정체성 한 문장

> 한 명의 대장장이가 제한된 하루 작업량 안에서 장비 한 점을 직접 탄생시키고, 강화 위험과 수식어 선택으로 고유한 운명을 만든 뒤, 그 작품이 세계에서 역사를 쌓으며 제작자의 명성과 다음 기회로 돌아오는 모바일 제작 게임.

### 1.2 대상 플레이어

- 짧고 즉각적인 모바일 조작을 선호하지만 결과물에는 장기 애착을 느끼고 싶은 플레이어
- 제작·수집·성장·확률 판단을 좋아하되 복잡한 직원·공장 운영은 원하지 않는 플레이어
- 최적 수치만이 아니라 장비의 출생, 성장 경로, 소유자와 사건 기록을 가치로 느끼는 플레이어

### 1.3 핵심 플레이어 약속

> 내가 직접 만든 장비는 판매 후 사라지는 소모품이 아니라 세상에 남아 활동하고, 그 장비가 만든 역사가 나를 더 유명한 대장장이로 만든다.

## 2. 기획의 필연성

### Why — 전달할 경험

평범한 재료가 플레이어의 기술, 선택, 욕심과 위험을 거쳐 고유한 작품이 되고, 그 작품이 세상에서 인정받는 경험을 전달한다.

### How — 경험을 만드는 전략

1. 제작 순간 영구 완성도 등급을 부여해 장비의 출생을 기록한다.
2. 강화 버튼을 누를 때마다 가치와 위험을 직접 판단하게 한다.
3. `+10` 단위 특수 강화에서 수식어의 계열·후보·전문화를 선택하게 한다.
4. 판매·납품된 장비를 세계에 잔존시켜 토너먼트·전쟁·모험 이력을 쌓게 한다.
5. 장비 성과를 골드뿐 아니라 명성, 관계, 의뢰, 희귀 재료와 후속 사건으로 환류한다.
6. 피로도와 날짜 진행으로 한 명의 대장장이가 무엇에 집중할지 선택하게 한다.

### What — 게임에 나타나는 결과

- 빠른 터치 제작과 묵직하게 성장하는 연출
- 무기·방어구·악세서리 제작
- 영구 완성도와 `+100` 강화
- `+10` 단위 일반 수식어 성장
- 고객·상인·토너먼트·전쟁
- 세계 잔존 장비와 제작 이력
- 통합 숙련도·계열 숙련도·설비 성장
- 피로도 기반 일일 운영

## 3. 뾰족한 재미

### 3.1 핵심 정의

> 한 점의 장비가 가벼운 재료에서 시작해 플레이어의 손, 선택과 위험을 거치며 고유한 작품으로 성장하고, 세상에서 새로운 역사를 만드는 과정을 지켜보는 재미.

### 3.2 감정 곡선

`가벼운 시작 → 작은 성공 → 첫 개성 탄생 → 애착과 욕심 증가 → 큰 위험 판단 → 작품 완성 → 세상의 평가 → 역사와 명성의 환류`

### 3.3 코어를 흐리는 방향

- 제작보다 직원·공장 운영이 중심이 되는 구조
- 의미 없는 장시간 연속 탭
- 선택 없이 확률 버튼만 반복하는 강화
- 높은 수치 장비만 남고 낮은 완성도 장비는 즉시 폐기되는 구조
- 판매 순간 장비와 영구 단절되는 구조
- 모든 전투를 직접 조작하거나 강제로 관전하는 구조
- 베팅이 제작보다 효율적인 주 성장 수단이 되는 구조

## 4. 핵심 루프

### 4.1 장비 한 점의 생애 루프

`목표·의뢰 확인 → 장비 종류·재료 선택 → 직접 제작 → 영구 완성도 결정 → 일반 강화 → +10 특수 강화·수식어 선택 → 계속 도전 또는 완성 → 판매·납품·후원 → 세계 활동 → 장비 이력·명성·보상 환류 → 더 중요한 작품 제작`

### 4.2 일일 운영 루프

`새 날짜·세계 소식 확인 → 오늘의 목표 선택 → 제작·강화·복원 → 피로도 판단 → 판매·납품·후원 → 하루 마치기 → 잔여 피로도 50% 이월 → 세계 사건·일정 진행`

### 4.3 행동·보상·진척

| 구분 | 내용 |
|---|---|
| 행동 | 제작 터치, 정밀 마감, 강화 1회 판정, 재료·촉매 선택, 수식어 후보 선택, 멈춤·추가 도전, 판매·납품·후원 |
| 즉시 보상 | 완성도, 강화 상승, 수식어, 외형·음향 변화, 판매가, 고객 반응 |
| 지연 보상 | 토너먼트·전쟁·모험 성과, 장비 별칭, 명성, 관계, 희귀 재료, 상위 의뢰 |
| 진척 | 대장장이 레벨, 장비 계열 숙련도, 설비, 제작법, 고객층, 대표작, 세계 기록 |

## 5. 플레이어 역할과 운영 제약

### 5.1 플레이어 역할

- 플레이어는 대장장이 한 명만 운용한다.
- 직원 고용·배치·급여·피로·숙련 관리 기능은 사용하지 않는다.
- 자동 단조는 직원 생산이 아니라 플레이어가 익힌 공정을 반복하는 설비 편의 기능이다.

### 5.2 피로도

- 제작·강화·중요 장비 복원은 피로도를 소비한다.
- 터치 횟수와 광클은 추가 피로도를 소비하지 않는다.
- 피로도 부족은 품질·강화 확률을 낮추지 않고 작업 시작만 제한한다.
- 날짜는 플레이어가 `하루 마치기`를 선택할 때만 진행된다.
- 작업 예약 기능은 사용하지 않는다.
- 남은 피로도의 50%를 다음 날 기본 피로도에 더한다.
- 별도 이월 상한은 두지 않지만, 사용하지 않고 계속 날짜를 넘기면 시작 피로도는 기본 최대치의 2배에 수렴한다.
- 자동 단조는 피로도 부족 시 중지하며 날짜를 자동 진행하지 않는다.

### 5.3 피로도 상대 비용

정확한 수치는 PoC 밸런스 변수로 둔다.

- 일반 강화 1회: 낮음
- 기본 장비 제작: 중간
- `+10` 특수 강화 1회: 중간~높음
- 중요 장비 복원: 높음

## 6. 제작 대상

### 6.1 최상위 장비 계열

1. 무기
2. 방어구
3. 악세서리

모든 계열은 공통 제작·완성도·강화 골격을 사용하고, 성능·수식어·세계 평가만 계열별로 달라진다.

### 6.2 장기 기본 종류

- 무기: 검, 도끼, 창, 둔기, 활, 지팡이, 단검
- 방어구: 흉갑, 투구, 장갑, 장화, 방패
- 악세서리: 반지, 목걸이, 부적, 브로치, 허리띠, 팔찌

### 6.3 초기 검증 종류

- 검: 공격과 강화 손맛 대표
- 흉갑: 방어와 생존 가치 대표
- 반지: 조건부·규칙형 효과 대표

생활·산업 도구와 소모품은 핵심 장비 루프가 검증되기 전까지 후속 확장 후보로만 보존한다.

## 7. 완성도 등급

### 7.1 역할

완성도는 기본 장비가 처음 완성될 때 붙는 영구 품질 접두어이며 장비가 어떻게 태어났는지를 기록한다.

`엉성한 → 평범한 → 정교한 → 뛰어난 → 완벽한`

### 7.2 판정 요소

`통합 대장장이 숙련도 + 장비 계열 숙련도 + 정밀 마감 결과 + 재료 가공 난이도 + 제한된 무작위`

### 7.3 불변 규칙

- 최초 제작 뒤 재연마·재단조로 완성도를 다시 판정하지 않는다.
- 낮은 완성도도 `+100`, 최고 수식어, 이벤트 참여에 접근할 수 있다.
- 완성도는 기본 공격·방어 수치보다 판매 가치, 명성, 평가, 외형·음향에 더 크게 작용한다.
- 완성도 때문에 낮은 등급 장비가 즉시 폐기물이 되어서는 안 된다.

## 8. 숙련도

혼합형 숙련도를 사용한다.

- 통합 대장장이 레벨: 전체 제작 확률, 재료 난이도 대응, 설비·제작법 해금
- 장비 계열 숙련도: 무기·방어구·악세서리별 완성도 분포와 희귀 후보 보정
- 세부 장비 종류: 별도 레벨 대신 제작법 해금과 제작 기록으로 관리

## 9. 강화와 수식어

### 9.1 공통 강화 골격

- 모든 장비는 최대 `+100`을 사용한다.
- 전 계열은 동일한 성공·유지·하락·파괴 규칙을 사용한다.
- `+1~+10`: 학습 구간, 실패 시 기본적으로 단계 유지
- `+11` 이상: 단계 하락 가능
- `+30` 이상: 장비 파괴 가능
- 실패 보정은 누적되고 성공 시 초기화된다.
- 안정 단조와 폭주 단조는 동일한 공통 계약을 사용한다.

### 9.2 일반 강화 입력

`강화 버튼 입력 → 비용·피로도 확인 → 1회 판정 → 결과 표시 → 다시 누르거나 멈춤`

- 버튼 한 번당 한 번의 강화 결과를 낸다.
- 보조재료·촉매·정밀 판정은 사용하지 않는다.
- 초기 연출은 빠르고 가볍다.
- 강화가 높아질수록 저음, 정적, 공명, 충격과 결과 대비가 점차 묵직해진다.

### 9.3 `+10` 특수 강화

`보조재료 선택 → 수식어 계열 결정 → 촉매 선택 → 위험 확인 → 정밀 강화 → 기본·희귀 후보 생성 → 최종 후보 선택`

혼합형 통제를 사용한다.

- 기본 후보는 재료와 현재 성장 경로를 통해 예측할 수 있다.
- 특정 조합이나 완벽 판정에서 희귀·숨은 후보가 추가된다.
- 보조재료가 계열을 정한다.
- 정밀 판정이 후보 수, 등급과 희귀 변형 확률을 바꾼다.
- 플레이어가 최종 후보를 선택한다.

### 9.4 수식어 성장 이정표

- `+10`: 첫 수식어 1티어
- `+20`: 첫 수식어 2티어·전문화
- `+30`: 두 번째 수식어 1티어
- `+40`: 두 번째 수식어 2티어·전문화
- `+50`: 세 번째 수식어 1티어
- `+60`: 세 번째 수식어 2티어·전문화
- `+70~+90`: 각 수식어 3티어
- `+100`: 세 수식어의 4티어와 전설 특성 완성

### 9.5 연출 성장 원칙

높은 강화의 묵직함은 단순히 연출 시간을 늘리는 것이 아니다.

- 망치 타격음의 저음과 잔향
- 성공 직전의 짧은 정적
- 무기·모루·용광로의 반응
- 불꽃의 양보다 밀도와 힘
- 수식어에 따른 외형·음향 변화
- 실패 손실과 다음 선택의 명확한 표시

`+10` 이정표는 일반 강화 한 번이 아니라 장비의 운명이 변하는 제작 의식으로 표현한다.

## 10. 세계 잔존 장비

### 10.1 기본 계약

판매·납품·후원된 장비는 특정 고객·세력의 소유가 되어 세계에 남는다.

`판매·납품 → 소유자 귀속 → 일반 활동 → 중요 사건 → 장비 이력 갱신 → 명성·관계·의뢰·보상 환류`

### 10.2 구현 범위 제한

실시간 세계 시뮬레이션 대신 이벤트 기반 기록 시스템을 사용한다.

장비별로 유지할 핵심 정보:

- 현재 소유자와 소속
- 현재 활동 유형
- 누적 주요 성과
- 최근 중요 사건
- 장비 상태
- 대장장이와의 관계

모든 NPC 위치, 매초 전투 계산, 세력 전체 전략 시뮬레이션은 구현하지 않는다.

### 10.3 일반 활동과 중요 이벤트

- 일반 활동: 게임 날짜 경과에 따라 간단한 보고로 처리
- 토너먼트·전쟁: 일정이 사전 공개되는 중요 이벤트
- 이벤트 개입: 핵심 NPC 직접 의뢰, 선택적 후원, 시장 수요 변화의 혼합형
- 한 이벤트의 주요 상호작용은 1~2개로 제한
- 결과 확인은 기본이며 관전과 게임 내 재화 베팅은 선택 기능
- 플레이어 직접 전투 조작은 사용하지 않는다.

### 10.4 명성 환류

- 즉시 명성: 완성도, 강화, 적합도, 수식어, 희귀성
- 사후 명성: 토너먼트 순위, 전쟁 공헌, 소유자의 생존·성공, 장기 사용과 역사적 사건
- 플레이어 통제 밖의 패배 때문에 기존 명성을 크게 차감하지 않는다.
- 실패는 재의뢰, 분석 정보, 수요 변화, 복원 사건으로 전환한다.

## 11. 대표작과 복원

### 11.1 대표작

혼합형 대표작을 사용한다.

- 시스템 대표작: 세계 성과로 자동 등록되는 역사적 장비
- 개인 대표작: 플레이어가 제한된 수만큼 직접 지정하는 애착 장비

대표작 지정은 전투 성능·강화 확률을 직접 높이지 않는다.

### 11.2 파손과 복원

- 일반 장비: 파손 시 활동 기록을 종료하고 역사만 보존
- 중요 장비: 대표작, 고강화, 희귀 수식어, 유명 소유자 등 조건을 만족하면 복원 의뢰 후보
- 복원 시 완성도, 강화 단계, 수식어, 별칭과 세계 이력을 유지
- 복원에서 추가 확률 파괴를 사용하지 않는다.
- 복원 비용과 난도는 희귀 재료, 피로도, 정밀 작업으로 표현한다.

## 12. 보호 경계

### 12.1 `INVARIANT`

- 한 명의 대장장이만 운용한다.
- 플레이어가 장비를 직접 제작한다.
- 장비는 최초 제작 시 영구 완성도 등급을 얻는다.
- 강화 버튼 한 번이 한 번의 결과와 멈춤·추가 도전 판단을 만든다.
- `+10` 단위에서 일반 수식어가 추가·성장·전문화된다.
- 강화가 높아질수록 가치·위험·연출의 무게가 증가한다.
- 판매된 장비는 세계에 남아 이력을 만든다.
- 장비 성과가 대장장이 명성과 다음 제작 기회로 돌아온다.
- 피로도가 하루 작업량을 제한하고 날짜 진행이 세계 사건을 움직인다.
- 조작은 빠르지만 작품과 결과는 묵직하게 느껴져야 한다.

### 12.2 `CHANGEABLE`

- 정확한 강화 확률·비용·피로도 수치
- 완성도별 수치 배율과 확률 분포
- 보관함 크기와 대표작 지정 수
- 세부 장비 종류와 출시 순서
- 수식어 이름·효과·후보 수
- 고객·세력·이벤트 빈도
- 관전 길이와 베팅 한도
- 장비 보고서 표현 방식

### 12.3 `REQUIRES_REAPPROVAL`

- 직원 또는 복수 대장장이 추가
- 직접 전투 조작 도입
- 완성도 재판정·승급 기능
- 장비 계열별 서로 다른 강화 실패 규칙
- 판매 장비의 세계 잔존 제거
- `+10` 수식어 이정표 구조 변경
- 피로도·날짜 시스템 제거
- 베팅을 핵심 성장 수단으로 전환

### 12.4 `OUT_OF_SCOPE`

- 직원 고용·배치·급여·피로 관리
- 복수 대장장이 운영
- 작업 예약·생산 대기열
- 플레이어 직접 전투
- 실시간 전체 세계 시뮬레이션
- 복잡한 장비 슬롯·세트 관리
- 소모품 중심 제작
- 공장형 무한 생산
- 완성도 재추첨
- 광고·과금 기반 무한 피로도 회복

## 14. PoC: 장비 한 점의 생애

### 14.1 검증 질문

> 플레이어가 직접 만든 장비에 애착을 느끼고, 그 장비가 세계에서 성과를 내는 결과를 다시 확인하고 싶어 하는가?

### 14.2 최소 플레이 흐름

`검투사 의뢰 → 철검 제작 → 완성도 결정 → +1~+9 일반 강화 → +10 특수 강화·수식어 선택 → 납품 → 승패 보고 → 장비 이력·명성 증가 → 같은 검투사의 재방문`

### 14.3 포함 범위

- 철검 1종
- 검투사 1명
- 완성도 5등급
- 통합 숙련도와 무기 계열 숙련도
- `+1~+10` 강화
- 화염·날카로움 수식어 2계열
- 피로도, 수동 날짜 진행, 잔여 피로도 50% 이월
- 장비 납품, 결과 보고, 세계 잔존 기록
- 골드, 명성, 관계, 재방문 의뢰

### 14.4 제외 범위

- 방어구·악세서리
- `+11~+100`
- 본격적인 단계 하락·파괴
- 자동 단조
- 상인 납품
- 전쟁·관전·베팅
- 대표작·복원
- 다수 고객과 복잡한 시장

### 14.5 성공 기준

1. 플레이어가 제작한 철검을 다른 철검과 구분해 기억한다.
2. 완성도와 수식어가 장비 성격을 다르게 느끼게 한다.
3. 강화 버튼을 다시 누를지 멈출지 고민하는 순간이 발생한다.
4. 납품 뒤 결과를 확인하고 싶은 기대가 생긴다.
5. 세계 결과가 단순 보상창이 아니라 장비 이력으로 느껴진다.
6. 검투사의 재방문이 다음 제작 동기를 만든다.
7. 피로도가 불쾌한 제한보다 하루 우선순위 선택으로 작동한다.
8. 플레이어가 단순히 더 높은 수치가 아니라 다른 이야기를 가진 장비를 다시 만들고 싶어 한다.

### 14.6 실패 조건

- 완성도가 색상이나 단순 능력치 배율로만 느껴진다.
- 일반 강화가 생각 없이 반복하는 버튼 작업이 된다.
- 수식어 후보의 용도 차이를 설명할 수 없다.
- 검투사 결과가 제작 선택과 무관하게 느껴진다.
- 납품 순간 장비에 대한 관심이 사라진다.
- 피로도 때문에 날짜만 반복해서 넘긴다.
- 낮은 완성도 장비가 즉시 폐기된다.
- 결과 보고보다 골드 숫자만 중요하게 느껴진다.

## 15. PoC 이후 재조정 절차

1. 외부 플레이테스트에서 성공·실패 기준별 행동·발언 증거를 수집한다.
2. 핵심 재미를 강화하지 않는 기능은 삭제·축소한다.
3. 제작, 강화, 세계 결과 중 기대가 끊기는 지점을 우선 수정한다.
4. PoC가 통과되면 `+30` 위험 강화 PoC로 하락·파괴·안정·폭주를 검증한다.
5. 이후 흉갑·반지로 공통 구조의 계열 확장성을 검증한다.
6. 토너먼트 일정, 전쟁, 선택적 관전, 대표작·복원 순으로 확장한다.

## 16. 프로덕션 SWOT·VRIO 게이트

SWOT와 VRIO는 현재 확정 판정이 아니라 PoC와 시장 조사 뒤 프로덕션 진입 여부를 결정하는 게이트다.

### 16.1 사전 SWOT 가설

**강점 가설**

- 제작·강화·세계 이력이 하나의 감정 루프로 연결된다.
- 전 장비 공통 규칙으로 학습 복잡도를 낮춘다.
- 빠른 모바일 입력과 장기 애착을 함께 노린다.
- 직원 없이 한 명의 대장장이 판타지에 집중한다.

**약점 가설**

- 장비 이력과 세계 사건 콘텐츠 제작량이 커질 수 있다.
- 확률 강화와 피로도가 함께 있을 때 피로감이 중첩될 수 있다.
- 낮은 등급 장비도 의미 있게 만드는 밸런스가 어렵다.
- `+100` 장기 곡선이 반복적으로 느껴질 위험이 있다.

**기회 가설**

- 제작 결과가 세계에서 살아남는 구조가 프로젝트의 식별 가능한 메시지가 될 수 있다.
- 토너먼트·전쟁 일정이 장비 수요와 재방문 동기를 만든다.
- 장비 이력과 대표작은 공유·회고·수집 콘텐츠로 확장 가능하다.

**위기 가설**

- 강화 중심 게임이나 방치형 경영 게임으로 오인될 수 있다.
- 관전·베팅·세계 시뮬레이션이 코어보다 커질 수 있다.
- 콘텐츠 양을 늘리는 과정에서 장비별 서사가 템플릿 반복으로 보일 수 있다.
- 확률·베팅 표현은 출시 지역과 플랫폼 정책 검토가 필요하다.

### 16.2 VRIO 판정 기준

| 항목 | 프로덕션 진입 기준 |
|---|---|
| Value | PoC에서 제작 선택이 세계 결과 기대와 재플레이 의도로 연결됨 |
| Rarity | 경쟁·유사작 조사에서 `장비 한 점의 세계 잔존 이력`이 명확한 차별 메시지로 확인됨 |
| Imitability | 단순 강화 수치가 아니라 제작·수식어·소유자·사건·명성이 연결된 데이터와 콘텐츠 구조를 구축할 수 있음 |
| Organization | 장비 데이터, 사건 템플릿, 밸런스 시뮬레이션, 테스트와 콘텐츠 제작 파이프라인을 소규모 팀이 지속 운영할 수 있음 |

PoC가 Value를 통과하지 못하면 Rarity·Imitability를 이유로 프로덕션을 진행하지 않는다. Organization이 통과하지 못하면 세계 사건과 장비 종류를 축소한 Vertical Slice로 다시 범위를 제한한다.
