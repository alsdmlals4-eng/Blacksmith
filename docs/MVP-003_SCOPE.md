## 목표

플레이어가 자신이 만든 철검을 기억하고, `+5`에서 납품할지 `+10`까지 더 도전할지 판단하며, 그 선택이 검투사의 지연된 결과와 장비 이력에 영향을 줬다고 이해하는지 검증한다.

## 플레이 흐름

```text
검투사 의뢰 확인
→ 철검 제작
→ 정밀 결과와 영구 완성도 결정
→ +5 기본 납품 조건 도달
→ 현재 납품 또는 +10 추가 도전
→ 납품
→ 수동 하루 종료
→ 지연 경기 결과
→ 장비 이력·명성·관계 갱신
→ 같은 검투사의 재방문
```

## 포함

- `iron_sword` 1종
- 검투사 카일 1명과 재방문 1회
- 제작 정밀 결과 3단계
- 영구 완성도 5등급
- `+0~+10` 강화
- `+5` 기본 납품 조건
- `+10` 선택적 추가 목표와 첫 수식어
- 화염·날카로움 수식어 계열
- 피로도 20, 수동 날짜 진행, 잔여 피로도 50% 이월
- 원자적 골드·재료·피로도·납품 처리
- 최소 하루 지연 결과
- 영구 세계 장비 기록
- 명성·관계·후속 의뢰
- PoC 행동 로그
- 세로 UI와 48dp 이상 주요 입력
- 정밀 보조 GOOD 경로와 모션 감소

## 제외

- `+11` 이상 하락과 `+30` 이상 파괴를 PoC 결과에 포함
- 자동 단조 참여
- 방어구·악세서리
- 상인·시장·세력
- 전쟁·관전·베팅
- 대표작·선택형 복원
- 직원·직접 전투·작업 예약
- 저장 마이그레이션
- 장기 콘텐츠·Production 수량 확정

## 설계 가드레일

- 버튼 입력 한 번당 일반 강화 판정 한 번을 유지한다.
- 피로도 부족은 작업 시작만 막고 성공률·완성도를 낮추지 않는다.
- 날짜는 사용자 입력으로만 진행한다.
- 낮은 완성도도 모든 PoC 강화·수식어·납품에 접근한다.
- 난수는 DEFEAT/WIN/DECISIVE_WIN 결과 밴드를 뒤집지 않는다.
- 보고서는 효과 선택, 부족 조건, 사건, 추가 이력을 분리한다.
- 정밀 입력에는 느린 포인터와 GOOD 범위 보조를 제공한다.
- 정밀 보조는 PERFECT 희귀 보너스를 자동 제공하지 않는다.
- 결과·위험은 색상 하나에만 의존하지 않는다.

- [ ] 사전조건 실패 시 모든 자원 무차감 실행 PASS
- [ ] 원자적 납품 실행 PASS

- [ ] 신규 테스트와 기존 Godot 회귀 전체 PASS

GitHub Actions 비용 게이트 이후 추가된 코드·Scene·신규 테스트는 아직 GitHub-hosted runner에서 실행하지 않았다. 사용자가 Actions 재사용 가능 상태를 알리면 `docs/CI_EXECUTION_POLICY.md` 순서로 위 검증을 실행한다.

## 플레이테스트 통과 신호

권장 표본은 외부 신규 플레이어 6명이다.

- 5명 이상이 자신의 철검을 구분해 기억
- 4명 이상이 +5/+10 선택 이유를 설명
- 4명 이상이 결과에 영향을 준 선택을 설명
- 4명 이상이 결과 보고 또는 재방문을 자발적으로 확인
- 낮은 완성도 즉시 폐기율 50% 미만
- 날짜 넘기기만 반복하는 세션 20% 미만
