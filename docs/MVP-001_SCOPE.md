# MVP-001 제작 터치 수직 프로토타입

## 목표

Android 세로 화면에서 철검 제작의 첫 손맛을 검증한다.

## 플레이 흐름

`철 선택 → 제작 시작 → 광클·자동 작업 → 피버 → 선택적 정밀 마감 → 철검 완성 → 강화 또는 다시 제작`

## 포함

- 철 주재료와 검 무기 종류의 고정된 첫 조합
- 터치당 제작 진행도
- 기본 자동 작업
- 연속 터치로 차오르는 피버 게이지
- 피버 중 터치·자동 작업 배율
- 피버 1회 이상 발동 시 최종 공격력 ×1.05·제작 가치 ×1.03, 반복 발동 비중첩
- ON/OFF 가능한 정밀 마감
- 움직이는 마감 포인터와 완벽·좋음·보통 판정
- 철검 원본 공격력은 20이며 보통 20(×1.00), 좋음 21(×1.05), 완벽 22(×1.10)로 실제 정수 공격력이 구분된다. 제작 가치는 각각 ×1.00·×1.05·×1.12다.
- 피버 적용 시 보통·좋음·완벽 공격력은 21·22·23, 합산 제작 가치는 ×1.03·×1.08·×1.15다.
- 원본 공격력·마감·피버·합산 제작 배율을 포함한 완성 결과 요약
- 완성 철검과 마감·피버 제작 효과를 MVP-002 강화 화면으로 전달
- 터치·마우스 공용 입력
- 상태 계산을 UI와 분리한 테스트 가능한 모델

## 제외

- 제작 재료 선택 확장
- 고객·상인 판매
- 저장·방치 복귀
- 사운드·진동·완성 아트
- Android export와 실기기 검증

## 완료 기준과 상태

- [x] 정밀작업 OFF로도 철검을 완성할 수 있다.
- [x] 빠르게 연속 터치하면 피버가 발동하고 작업량이 증가한다.
- [x] 정밀작업 ON에서는 타이밍에 따라 마감 품질이 달라진다.
- [x] 완성 결과를 강화 화면으로 전달하거나 다시 제작할 수 있다.
- [x] 좋은·완벽한 마감이 실제 기본 공격력과 제작 가치에 반영된다.
- [x] 원본 공격력과 마감·피버·합산 제작 배율을 분리해 강화·보관까지 전달한다.
- [x] 피버를 한 번 이상 발동하면 결과 보너스가 적용되고 반복 발동은 중첩되지 않는다.
- [x] 모델 단위 검증·제작→강화 통합 검증·데이터 정적 검증이 통과한다.
- [ ] 실제 화면 렌더와 플레이 감각을 검수한다.
- [ ] Android 실기기에서 세로 화면과 터치 입력을 검증한다.
