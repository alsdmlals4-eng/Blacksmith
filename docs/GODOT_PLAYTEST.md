# Godot 제작·강화·장비 생애 테스트 실행

## 필요한 버전

- Godot 4.7.1 stable
- 기존 기준선: `main`
- MVP-003 구현 후보: `agent/implement-equipment-lifecycle-poc`

## 사용자가 할 일

1. GitHub Desktop에서 Blacksmith 저장소를 선택한다.
2. `Fetch origin` 후 PR #35 브랜치 `agent/implement-equipment-lifecycle-poc`로 전환한다.
3. `Pull origin`을 누른다.
4. Godot에서 저장소 루트의 `project.godot`을 연다.
5. `scenes/main/main.tscn`을 열고 F6를 누른다.

현재 `project.godot` 기본 실행 Scene은 강화 테스트 Scene이므로 전체 흐름은 `main.tscn` F6로 실행한다.

## 장비 생애 PoC 진입

기존 Prototype 화면 왼쪽 위의 **장비 생애 PoC** 버튼을 누르거나 `scenes/test/equipment_lifecycle_poc.tscn`을 열고 F6를 누른다.

확인 흐름:

```text
검투사 카일 의뢰 수락
→ 철검 제작
→ 마감 정타와 영구 완성도 확인
→ 일반 강화 버튼당 판정 1회
→ +5에서 납품 또는 +10 추가 도전
→ 수동 하루 마치기
→ 지연 경기 결과
→ 기여·부족 조건과 장비 이력 확인
→ 같은 카일의 재방문
```

### PoC 작업량·날짜

- 시작 작업량이 20인가
- 철검 제작 시작 시 3만 감소하고 망치질 탭에는 추가 차감이 없는가
- 일반 강화 한 번에 1이 감소하는가
- +10 특수 강화 시작에 3이 감소하는가
- 자원이 부족하면 골드·재료·작업량이 모두 변하지 않는가
- 작업량 부족 시 날짜를 자동으로 넘기지 않는가
- `하루 마치기`를 눌렀을 때만 날짜가 진행되는가
- 다음 날 작업량이 `20 + floor(남은 작업량 × 0.5)`인가

### +5 / +10 판단

- +4에서는 납품 버튼이 비활성화되는가
- +5에서는 납품 가능하며 예상 결과와 적합도 근거가 보이는가
- +10 도전은 선택 사항으로 남는가
- 강화 화면 내부에서 +10을 초과할 수 없는가
- 특수 강화 정밀 타격 중에는 대장간 복귀가 차단되는가
- +10 성공 뒤 수식어가 장비 기록에 남는가

### 세계 결과·재방문

- 납품 직후 결과가 즉시 뜨지 않는가
- 최소 하루 뒤 결과가 생성되는가
- 결과 화면이 자동으로 닫히지 않는가
- `효과가 있었던 선택 / 부족했던 조건 / 기여 점수`가 텍스트로 표시되는가
- 납품 장비의 소유자·결과·이력이 유지되는가
- 보고를 열면 명성·관계가 한 번만 반영되는가
- 같은 카일이 이전 장비를 언급하며 재방문하는가

### 정밀 입력·접근성

- 모든 주요 버튼의 터치 영역이 약 48dp 이상인가
- **정밀 보조 · GOOD 범위**를 켜면 GOOD 경로를 사용하고 PERFECT가 자동 제공되지 않는가
- **모션 감소 · 느린 포인터**를 켜면 제작·특수 강화 포인터가 느려지는가
- 성공·실패·위험이 색상 외 텍스트로도 구분되는가
- 작은 세로 화면에서 겹침·잘림·가로 스크롤이 없는가

## 기존 제작 품질 확인

- `scenes/main/main.tscn`을 열고 F6로 전체 제작 흐름을 실행한다.
- 화면 버전이 `POC v0.6.4 · main · 2026.07.23.1`인지 확인한다.
- 보통 마감: 원본 공격력 20 → 품질 적용 20, 제작 가치 ×1.00
- 좋은 마감: 원본 공격력 20 → 품질 적용 21, 제작 가치 ×1.05
- 완벽한 마감: 원본 공격력 20 → 품질 적용 22, 제작 가치 ×1.12
- 피버 1회 이상이면 공격력 ×1.05·제작 가치 ×1.03이 한 번만 더해지는가
- 여러 번 발동해도 결과 보너스가 중첩되지 않는가
- 완벽 마감+피버가 공격력 23·제작 가치 ×1.15인가
- 강화 화면과 보관함에서 원본·마감·피버·합산 제작 배율이 유지되는가
- 반복 자동 단조의 두 번째 무기부터는 보통 마감·피버 미적용인가

## 강화 분류

### 일반 강화

- +10 단위가 아닌 모든 단계
- 버튼 한 번으로 즉시 성공·실패 판정
- 보조재료·촉매·정밀 판정 미사용
- 이전 특수 강화 선택 효과 비누출

### 특수 강화

- +10·+20·…·+100
- 보조재료·촉매 선택
- 정밀 판정
- 수식어 추가 또는 티어 성장

## 위험·성장 확인

- 공격력과 판매가가 현재 강화 적용 기준으로 점진 증가하는가
- 고단계일수록 다음 효과·판매가·비용 증가가 커지는가
- 성공·유지·하락·파괴 확률이 표시되는가
- +11부터 단계 하락이 발생할 수 있는가
- +30부터 무기 파괴가 발생할 수 있는가
- 단계 유지·하락 실패에서 실패 보정이 누적되고 성공 시 초기화되는가
- 선택한 특수 강화 재료가 결과와 무관하게 시도 시작 시 소비되는가
- 파괴 결과가 보관 불가·세션 종료로 이어지는가
- 실패 정책·확률·위험표는 `enhancement_balance.json`, 수식어 이정표는 `enhancement_milestones.json`만 소유하는가
- 안정 단조와 보호 촉매가 표시된 위험을 실제로 줄이는가

## 폭주 단조 확인

- 성공 시 낮은 확률로 총 2단계 상승하는가
- 도약이 없으면 1단계 상승하는가
- +10·+20·… 특수 강화에서 선택할 수 없는가
- +9·+19·+29·…에서 선택할 수 없는가
- 어떤 경우에도 특수 강화 수식어 이정표를 건너뛰지 않는가

## 수식어 성장

- +10: 첫 수식어 1티어
- +20: 첫 수식어 2티어
- +30: 두 번째 수식어 1티어
- +40: 두 번째 수식어 2티어
- +50: 세 번째 수식어 1티어
- +60: 세 번째 수식어 2티어
- +70·+80·+90: 각 수식어 3티어
- +100: 모든 수식어 4티어

## 보관함·경제·자동 단조 확인

- 강화 도중 `강화 종료 및 보관`이 가능한가
- 최대 6개까지만 보관되는가
- 파괴 무기는 보관되지 않는가
- 일반 강화를 한 번 누르면 표시된 비용만큼 골드가 감소하는가
- 특수 강화에서 보조재료와 촉매가 시도 시작 시 각각 1개 감소하는가
- 골드가 부족하면 강화 판정·실패 보정·누적 시도 횟수가 변하지 않는가
- 자동 단조 목표 도달 시 보관되는가
- 보관함 가득 참·골드 부족·수동 중지에서 종료하는가
- 파괴 시 반복 설정에 따라 새 무기로 재시작하거나 종료하는가
- 장시간 실행 중 멈춤·무한 반복·심한 프레임 저하가 없는가
