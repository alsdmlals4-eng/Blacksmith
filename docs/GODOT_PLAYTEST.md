# Godot 강화 테스트 실행

## 필요한 버전

- Godot 4.7.1 stable
- 프로젝트 브랜치: `main`

## 사용자가 할 일

1. GitHub Desktop에서 Blacksmith 저장소를 선택한다.
2. `Fetch origin` 후 `Pull origin`을 누른다.
3. Godot에서 저장소 루트의 `project.godot`을 연다.

## 강화만 바로 테스트

1. FileSystem에서 `scenes/test/enhancement_test.tscn`을 연다.
2. `F6` 또는 우측 상단의 **현재 장면 실행**을 누른다.
3. 테스트용 철검 +0 상태에서 +100까지 강화한다.

## 강화 분류

### 일반 강화

- +10 단위가 아닌 모든 단계
- 버튼 한 번으로 즉시 성공·실패 판정
- 보조재료 사용 안 함
- 촉매 사용 안 함
- 정밀 판정 사용 안 함

### 특수 강화

- +10·+20·+30·…·+100
- 보조재료 선택
- 촉매 선택
- 정밀 판정 진행
- 수식어 추가 또는 티어 성장

```text
+1~+9 일반 강화
→ +10 특수 강화
→ +11~+19 일반 강화
→ +20 특수 강화
→ 같은 구조를 +100까지 반복
```

보조재료와 촉매는 화면만 숨기는 것이 아니라 +10 단위 특수 강화 시도에만 실제 효과가 적용된다. +5나 일반 단계에서는 이전에 선택했던 보조재료·촉매 효과와 정밀 판정 보너스가 적용되지 않는다.

## 수식어 성장 이정표

- +10: 첫 수식어 1티어 추가
- +20: 첫 수식어 2티어
- +30: 두 번째 수식어 1티어 추가
- +40: 두 번째 수식어 2티어
- +50: 세 번째 수식어 1티어 추가
- +60: 세 번째 수식어 2티어
- +70: 첫 수식어 3티어
- +80: 두 번째 수식어 3티어
- +90: 세 번째 수식어 3티어
- +100: 모든 수식어 4티어

성공률은 10단계 패턴을 반복하되 높은 강화 구간일수록 기본 성공률이 조금씩 낮아진다. 실패 시 단계는 유지되고, 기존 실패 보정은 계속 적용된다.

## 전체 제작 흐름 테스트

`scenes/main/main.tscn`을 열고 `F6`을 누르면 다음 흐름을 확인할 수 있다.

```text
철검 제작 → 광클·피버 → 정밀 마감 → 일반 강화 → 특수 강화 → +100
```

## 키보드

- Space: 현재 가능한 망치질·일반 강화·특수 강화 타격 실행
- 마우스 클릭과 모바일 터치도 동일하게 지원

## 확인 항목

- +1~+9가 타이밍 화면 없이 원클릭으로 판정되는가
- +5에서도 정밀 판정이 나타나지 않는가
- +10 단위에서만 `[특수 강화]` 화면이 나타나는가
- 특수 강화에서만 보조재료·촉매 선택 영역이 나타나는가
- 특수 강화에서만 정밀 판정이 진행되는가
- 일반 강화에 이전 촉매 보너스가 적용되지 않는가
- 실패해도 단계가 내려가거나 무기가 파괴되지 않는가
- +10부터 수식어가 추가·강화되는가
- +100에서 강화가 완료되고 세 수식어가 4티어가 되는가
- 720×1280 세로 화면에서 스크롤과 버튼이 정상 동작하는가
