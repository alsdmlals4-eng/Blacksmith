# Godot POC 실행·검수

## 사용자가 할 일

1. GitHub Desktop에서 Blacksmith 저장소를 `Fetch origin → Pull origin`
2. Godot 4.7.1로 저장소 루트의 `project.godot` 열기
3. `F5`

기본 Main Scene은 `scenes/test/enhancement_test.tscn`이므로 강화 테스트가 즉시 열린다.

## 전체 제작→강화

`scenes/main/main.tscn`을 현재 Scene으로 실행하면 다음 흐름을 확인한다.

```text
철검 제작
→ 광클·자동 작업·피버
→ 선택적 정밀 마감
→ 일반·특수 강화
→ 보관함
→ 자동 단조
```

## 강화 규칙

### 일반 강화

- +10 단위가 아닌 모든 단계
- 원클릭 즉시 판정
- 보조재료·촉매·정밀 판정 미사용
- 이전 특수 강화의 재료 효과가 누출되지 않음

### 특수 강화

- +10·+20·…·+100
- 보조재료·촉매 선택
- 정밀 판정
- 수식어 추가 또는 티어 성장

### 실패·위험

- +1~+10: 실패 시 단계 유지
- +11부터: 단계 하락 가능
- +30부터: 무기 파괴 가능
- 실패당 성공률 +4%p, 최대 +24%p
- 성공 시 실패 보정 초기화
- 안정 단조와 수호 가루는 파괴 위험을 막을 수 있음

### 폭주 단조

- 사용할 수 있는 일반 강화 성공 시 8% 확률로 총 2단계 상승
- 특수 강화에서는 사용 불가
- 도약으로 +10 단위 특수 강화를 건너뛰는 위치에서도 사용 불가
- 자동 단조 목표를 넘길 수 있으면 균형 단조로 전환

## 수식어 이정표

- +10: 첫 수식어 1티어
- +20: 첫 수식어 2티어
- +30: 두 번째 수식어 1티어
- +40: 두 번째 수식어 2티어
- +50: 세 번째 수식어 1티어
- +60: 세 번째 수식어 2티어
- +70·+80·+90: 각 수식어 3티어
- +100: 모든 수식어 4티어

## 자동 단조 확인

- 목표 단계를 지정할 수 있는가
- 목표 도달 후 보관함에 자동 저장되는가
- 반복을 켜면 6칸이 찰 때까지 새 철검을 생산하는가
- 골드와 선택 재료가 실제 감소하는가
- 보조재료·촉매 재고가 0이면 빈 슬롯으로 계속하는가
- 목표 직전 폭주 단조가 2단계 초과하지 않도록 전환되는가
- 골드 부족·파괴·중지 요청·보관함 가득 참 상태가 명확한가

## 화면 확인

- 현재 공격력, 다음 성장량, 성공 후 가격, 시도 비용, 유지·하락·파괴 확률
- 촉매 가격·효과와 특수 강화 예상 효과
- 일반/특수 강화 화면 구분
- 보관함 무기 정보와 예상 손익
- 720×1280에서 버튼·스크롤·버전 배지
- 작은 화면에서 텍스트 겹침·잘림·터치 대상 크기

자동 테스트 PASS는 실제 화면·손맛·Android 터치 PASS가 아니다. 실제 Godot 창, Android 기기, AAB는 각각 별도 증거로 기록한다.
