# Decision Log

- 상태: 확정

- 재검토: 실제 기기에서 입력 피로·화면 구조 문제가 확인될 때

## DEC-002 제작 입력

- 상태: 확정·구현
- 결정: 빠른 연속 터치는 피버를 채우며 과열 불이익을 주지 않는다.
- 제작 정밀 마감은 선택 가능하다.

## DEC-003 무기 성장

- 상태: 확정·구현
- 결정: 기본 제작 결과는 수식어가 없고, +10 단위 특수 강화에서 수식어를 추가·성장시킨다.
- 최대 강화: +100

## DEC-004 재료와 숨은 무기

- 상태: 확정
- 결정: 주재료·보조재료·촉매를 사용하며 보조재료·촉매는 특수 강화에서만 적용한다.
- 숨은 무기 변환: 미구현

## DEC-005 판매

- 상태: 확정·미구현
- 결정: 고객 방문은 고정 기본금+결과 인센티브, 상인 납품은 지정 조건+즉시 가산금이다.

## DEC-006 단순성

- 상태: 확정
- 결정: 직원과 무기 수리를 초기 범위에서 제외한다.

## DEC-007 프로토타입 분할

- 상태: 확정
- MVP-001 제작
- MVP-002 강화·보관·자동 단조
- MVP-003 방문 검투사 판매
- MVP-004 상인 납품

## DEC-008 초기 실패 보호

- 상태: 대체됨
- 이전 결정: 실패 시 단계 유지, 파괴 없음, 실패당 +5%p·최대 +20%p
- 대체 이유: 고단계 성공 가치와 위험 선택의 긴장감이 부족해 POC v0.5.0에서 위험 구조로 확장
- 현재 정본: DEC-012와 `data/crafting/enhancement_balance.json`

## DEC-009 수식어 판정

- 상태: 확정·구현
- 숫돌 → `날카로운`
- 화염석 → `불타는`
- 정령의 심장 → `정령이 깃든`
- +10/+30/+50에서 새 수식어, 이후 이정표에서 티어 성장, +100 전체 4티어

## DEC-010 강화 분류

- 상태: 확정·구현
- 일반 강화: +10 단위가 아닌 모든 단계, 원클릭
- 특수 강화: +10·+20·…·+100, 재료·촉매·정밀 판정
- 일반 강화에는 이전 특수 강화 효과가 적용되지 않는다.

## DEC-011 초기 +100 성공률 곡선

- 상태: 대체됨
- 이전 결정: 구간당 -2%p, 최대 -18%p, 최저 35%
- 현재 결정: 구간당 -2.5%p, 최대 -25%p, 최저 25%
- 현재 정본: `data/crafting/enhancement_balance.json`

## DEC-012 고단계 위험과 실패 보정

- 상태: 확정·구현
- +1~+10 실패: 단계 유지
- +11부터: 단계 하락 가능
- +30부터: 파괴 가능
- 실패당 성공률 +4%p, 최대 +24%p
- 하락해도 실패 보정 유지, 성공 시 초기화
- 이유: 고단계 성공의 가치와 안전/도박 선택을 명확히 만든다.
- 재검토: 실제 플레이에서 파괴 스트레스·복구 시간·경제 손실을 측정할 때

## DEC-013 점진 성장·가속 가격

- 상태: 프로토타입·구현
- 강화 효과는 고정 덧셈이 아니라 현재 공격력 기준 비율로 증가한다.
- 높은 구간에서 성장률·비용·판매가 증가 폭을 키운다.
- 다음 결과와 위험을 시도 전에 표시한다.
- 실제 수치 정본: `data/crafting/enhancement_balance.json`

## DEC-014 단조 기술

- 상태: 프로토타입·구현
- 균형 단조: 기본
- 안정 단조: 비용 증가, 성장 감소, 파괴 방지, 하락 위험 감소
- 폭주 단조: 성공 시 8% 확률로 총 2단계 상승
- 폭주는 특수 강화와 특수 강화를 건너뛸 수 있는 위치에서 사용 불가

## DEC-015 보관함·자동 단조

- 상태: 프로토타입·구현
- 보관함 용량: 6
- 목표 단계 도달 시 자동 보관
- 반복 시 보관함이 찰 때까지 새 무기 생산
- 골드·재료를 소비하며 지정 재료 재고가 없으면 빈 슬롯으로 진행
- 현재 자원·보관함은 메모리 전용
- 재검토: 저장·경제 구현 시 데이터 소유권과 복귀 정책 확정

## DEC-016 Base 운영체계

- 상태: 확정·적용
- Base 기준 커밋을 프로젝트에 고정하고 원격 최신을 암묵 적용하지 않는다.
- Work Mode와 trigger 기반 최소 Skill을 자동 선택한다.
- 기존 프로젝트는 audit·보존·verify 없이 대량 이동·삭제하지 않는다.
- 정본 변경은 Registry·소비자·테스트·Workflow 참조 최신성을 함께 검수한다.
