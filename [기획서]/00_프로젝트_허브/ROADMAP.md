# Roadmap

## Concept — 통과

완료:
- 제작·강화·판매 루프를 한 문장으로 설명할 수 있다.
- 초기 제외 범위가 기록돼 있다.
- Android 모바일·Google Play·Godot 기술 기준이 확정돼 있다.

## Prototype — 현재

### MVP-001 제작 터치 — 구현, 자동 검증 통과

`철 고정 → 광클·자동 제작 → 피버 → 선택적 정밀 마감 → 철검 완성`

완료:
- 터치 한 번마다 제작 반응이 즉시 계산된다.
- 빠른 연속 터치로 피버가 발동하고 작업 배율이 적용된다.
- 정밀작업 OFF로도 철검을 완성할 수 있다.
- 정밀작업 ON에서 완벽·좋음·보통 마감이 구분된다.
- Godot 파싱과 제작 모델 테스트가 통과한다.

남음:
- 실제 화면 렌더와 Android 터치 검증

### MVP-002 강화 이정표 — 구현, 자동 검증 통과

`철검 → 보조재료·촉매 선택 → 선택적 정밀 강화 → 성공·실패 → +5 → 첫 수식어`

완료:
- +1~+5 단계별 성공률을 표시한다.
- 숫돌·화염석·정령의 심장 성질을 강화 시도마다 누적한다.
- 살라맨더의 핵 촉매가 성공률을 높인다.
- 정밀 강화 GOOD·PERFECT가 성공률을 높인다.
- 실패 시 단계 유지, 무기 파괴·수리 없음, 다음 성공률 보정이 발생한다.
- +5에서 누적 재료 성질에 맞는 1티어 수식어가 붙는다.
- 제작 완료 후 강화 화면으로 이동하고 +5 후 새 제작으로 복귀한다.
- Godot 파싱과 강화 모델 테스트 5건이 통과한다.

남음:
- 실제 강화 화면·스크롤·상태 전환 시각 검수
- Android 실기기 정밀 강화 터치 검증

### MVP-003 첫 판매 — 다음

`수식어 무기 선택 → 방문 검투사 요청 확인 → 고정 기본금 → 경기 결과 → 인센티브·명성·단골 확률`

종료 기준:
- 방문 검투사가 상황과 목적을 제시한다.
- 플레이어가 보유 무기 중 하나를 자유롭게 선택한다.
- 기본 대금은 선택 무기와 관계없이 고정된다.
- 무기 적합성과 경기 결과에 따라 추가 인센티브가 계산된다.
- 플레이어 무기로 승리하면 명성·인기·단골 확률이 오른다.
- 상인 납품과 역할이 겹치지 않는다.
- 결과 계산과 화면 흐름에 자동 테스트가 있다.

### MVP-004 상인 납품

`지정 무기 의뢰 → 제작·강화 → 조건 비교 → 즉시 납품 → 강화·수식어 가산금`

종료 기준:
- 상인이 원하는 무기 종류를 명확히 지정한다.
- 강화 단계와 수식어가 즉시 판매가에 반영된다.
- 고객 방문 결과형 판매와 구분된다.

## Graybox

- 대장간, 상인, 고객 화면 전환
- 기본 저장·복귀와 방치 보상
- 검투사·군대·모험가의 최소 결과 시뮬레이션
- 상인 구매 상점

## First Playable

- 20~30분의 초기 성장
- 기본 무기와 재료 해금
- +15까지 강화 이정표
- 숨은 무기 조합 1종
- 실패·복귀·전쟁 시즌 종료 흐름

## Vertical Slice

- 출시 목표 UI 품질의 대장간 한 화면
- 검투사 경기 1종
- 군대 방어 1개 시즌
- 모험가 탐험 지역 1곳
- Google Play 내부 테스트용 AAB
