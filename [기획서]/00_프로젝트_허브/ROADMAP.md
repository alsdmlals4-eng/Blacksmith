# Roadmap

## Concept — 통과

완료:
- 제작·강화·판매 루프를 한 문장으로 설명할 수 있다.
- 초기 제외 범위가 기록돼 있다.
- Android 모바일·Google Play·Godot 기술 기준이 확정돼 있다.

## Prototype — 현재

### MVP-001 제작 터치 — 구현, 검증 중

`철 고정 → 광클·자동 제작 → 피버 → 선택적 정밀 마감 → 철검 완성 → 다시 제작`

종료 기준:
- 터치 한 번마다 제작 반응이 즉시 보인다.
- 빠른 연속 터치로 피버가 발동하고 작업 배율이 적용된다.
- 정밀작업 OFF로도 철검을 완성할 수 있다.
- 정밀작업 ON에서 완벽·좋음·보통 마감이 구분된다.
- 모델 자동 테스트와 실제 화면·Android 터치 검증을 통과한다.

### MVP-002 강화 이정표 — 다음

`철검 → 강화 재료 선택 → 피버·정밀 강화 → +5 → 예리함 계열 첫 수식어`

종료 기준:
- 강화 확률과 보정 원인을 화면에서 이해할 수 있다.
- +5에서 보조재료 성질에 맞는 수식어가 붙는다.
- 실패 시 무기가 파괴되지 않고 재료만 소모된다.

### MVP-003 첫 판매

`완성 무기 선택 → 방문 검투사에게 지급 → 기본금 → 경기 결과 → 인센티브·명성`

종료 기준:
- 고객 직접 방문 판매 한 건이 결과 인센티브까지 완료된다.
- 상인 납품과 역할이 겹치지 않는다.

## Graybox

- 대장간, 상인, 고객 화면 전환
- 기본 저장·복귀와 방치 보상
- 검투사·군대·모험가의 최소 결과 시뮬레이션
- 상인 납품 의뢰

## First Playable

- 20~30분의 초기 성장
- 기본 무기와 재료 해금
- +15까지 강화 이정표
- 숨은 무기 조합 1종
- 실패·복귀·전쟁 시즌 종료 흐름

## Vertical Slice

- 출시 목표 UI 품질의 대장간 한 화면
- 검투사 경기 1종
- 군대 방어 1개 시즌
- 모험가 탐험 지역 1곳
- Google Play 내부 테스트용 AAB
