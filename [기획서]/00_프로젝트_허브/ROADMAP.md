# Roadmap

## Concept — PASS

완료:
- 제작·강화·보관·판매 루프를 한 문장으로 설명할 수 있다.
- Android 모바일·Google Play·Godot 기술 기준이 확정돼 있다.
- 직원·수리·결제·광고·서버는 현재 범위에서 제외돼 있다.
- 일반 강화와 +10 단위 특수 강화의 역할이 분리돼 있다.

## Prototype — CURRENT

### MVP-001 제작 — 구현, 자동 검증 PASS 이력

`철 고정 → 광클·자동 제작 → 피버 → 선택적 정밀 마감 → 철검 완성 → 강화 진입`

완료:
- 보통·좋음·완벽 마감의 실제 정수 공격력 20·21·22와 제작 가치 효과 연결
- 원본 공격력·품질 적용 공격력·가치 배율의 강화·보관 전달
- 피버 1회 이상 결과 공격력 ×1.05·제작 가치 ×1.03과 반복 발동 비중첩
- 원본 공격력·마감·피버·합산 제작 배율의 강화·보관 전달
- 반복 자동 단조의 새 철검 보통 마감·피버 미적용 고정

남음:
- 실제 화면 시각 검수
- Android 터치·세로 비율·안전 영역 검증

### MVP-002 +100 강화·보관 — 구현, 자동 검증 PASS 이력

`철검 → 일반 강화 → +10 단위 특수 강화 → 성장·하락·파괴 → 수식어 성장 → 수동 보관`

완료:
- 현재 공격력 기반 점진 성장과 고단계 가격·비용 가속
- 일반/특수 강화 분리와 재료 효과 비누출
- +100 수식어 3개·4티어 성장
- 실패 보정, 단계 하락, 파괴
- 균형·안정·폭주 단조
- 다음 효과·가격·위험 표시
- 최대 6개 보관함과 상세 정보
- 수동 강화의 실제 골드 차감
- 수동 특수 강화의 선택 보조재료·촉매 실제 차감
- 골드·선택 재료 부족 시 무차감·무판정 차단
- 수동·자동 강화가 동일한 공유 경제 거래를 사용
- 강화 실패·확률·위험 수치의 단일 정본을 `enhancement_balance.json`으로 통합
- `enhancement_milestones.json`에서 폐기된 실패 정책 제거, 수식어 이정표만 유지
- 최대 +100에서 도달 가능한 위험 decade 0~9만 유지
- 성공률 패턴·보정·위험 단조성·이정표·재료 참조 의미 검증 추가

남음:
- +100 수동 완주·피로도·위험 보상 관찰
- Android 특수 강화 타이밍·스크롤 검증
- 실제 경제 획득 루프 연결

### MVP-002A 자동 단조 — 구현, 자동 검증 PASS 이력

`목표·방식·특수 재료 지정 → 빠른 자동 강화 → 목표 도달 → 자동 보관 → 보관함 가득 찰 때까지 반복`

완료:
- 목표 단계·반복·단조 방식 지정
- 특수 강화 보조재료·촉매 지정
- 수동 강화와 동일한 골드·재료 거래 사용
- 지정 재료 부족 시 빈 슬롯 fallback 후 동일 거래 경로로 진행
- 골드 부족·보관함 가득 참·수동 중지
- 파괴 시 반복 정책
- 폭주 2단계 도약과 특수 강화 건너뛰기 차단

남음:
- 장시간 반복 성능·무한 루프·메모리 검증
- 저장·복귀 정책
- 자동 정밀 판정과 수동 플레이 보상의 역할 분리
- 자동화가 수동 광클과 선택의 재미를 약화시키지 않는지 플레이테스트

### MVP-003 방문 검투사 판매 — 정합성 개선 후 NEXT

`보관 무기 선택 → 방문 검투사 목적 확인 → 고정 기본금 → 경기 결과 → 인센티브·명성·단골 확률`

종료 기준:
- 방문 검투사가 상황과 목적을 제시한다.
- 보관함 무기 중 하나를 선택한다.
- 기본 대금과 성능·수식어 기반 인센티브가 분리된다.
- 경기 결과가 명성·인기·단골 확률에 반영된다.
- 상인 즉시 납품과 역할이 겹치지 않는다.
- 정상·부적합·패배·보관함 빈 상태 테스트가 있다.

### MVP-004 상인 납품

`지정 무기 의뢰 → 제작·강화 → 조건 비교 → 즉시 납품 → 강화·수식어 가산금`

종료 기준:
- 지정 무기·최소 강화·수식어 조건이 명확하다.
- 강화 단계와 수식어가 즉시 판매가에 반영된다.
- 고객 방문 결과형 판매와 구분된다.
- 재료·골드 획득이 자동 단조 소비와 연결된다.

## Graybox

- 대장간·보관함·상인·고객 화면 전환
- 재화·재료 재고와 최소 상점
- 저장·복귀와 방치 보상
- 검투사·군대·모험가의 최소 결과 시뮬레이션
- Android 실제 기기 핵심 루프 검증

## First Playable

- 20~30분 초기 성장 세션
- 기본 무기·재료 해금
- 자동 단조와 수동 플레이의 역할 분리
- 숨은 무기 조합 1종
- 실패·파괴·복구·판매 흐름
- 저장·복귀 안정성

## Vertical Slice

- 출시 목표 UI 품질의 대장간·강화·보관·판매 대표 구간
- 검투사 경기 1종
- 외부 플레이테스트와 행동 증거
- 접근성 장벽·대안 검수
- 대표·최악 장면 성능 예산 검증
- 제작 파이프라인 재현
- Google Play 내부 테스트용 AAB

- 저장·복귀

- 외부 플레이 또는 명시된 표본의 플레이테스트
- 접근성·성능 기준과 실측
- 자동 단조의 장기 경제·피로도 검증
