# Roadmap

## Concept — PASS

- Android 모바일·Google Play·Godot 기준
- 광클 피버 제작
- 일반/특수 강화와 수식어 성장
- 고객 방문/상인 납품 판매 구분
- 직원·무기 수리 제외

## Prototype — CURRENT

### MVP-001 제작 — 구현·자동 검증

```text
철 고정 → 광클·자동 작업 → 피버 → 선택적 정밀 마감 → 철검 → 강화
```

남음: 실제 화면·Android 터치.

### MVP-002 강화·보관·자동 단조 — 구현·자동 검증

```text
일반 강화 9회 리듬
→ 특수 강화
→ 성장·가격·하락·파괴
→ 안전 또는 폭주 선택
→ 보관
→ 목표 단계 자동 반복
```

완료:

- +100 일반/특수 강화
- 다음 효과·가격·비용·확률
- +11 하락·+30 파괴·실패 보정
- 촉매와 균형·안정·폭주 단조
- 폭주 8% 2단계 도약 제한
- +100 수식어 성장
- 6칸 보관함
- 골드·재료를 쓰는 자동 단조
- 모델 테스트 12건

남음:

- 실제 화면·피로·고위험 체감
- Android 터치·스크롤·안전 영역
- 저장·복귀와 실제 경제

### MVP-003 방문 검투사 판매 — NEXT

```text
보관 무기 선택
→ 검투사 요청 확인
→ 고정 기본금
→ 경기 결과
→ 인센티브·명성·단골
```

종료 기준:

- 요청 상황과 목적 표시
- 보관함 무기 자유 선택
- 고정 기본금과 적합도 기반 결과 분리
- 경기 인센티브·명성·단골 계산
- 파괴된 무기·빈 보관함·부적합 무기 경계 처리
- 자동 테스트와 결과 로그

### MVP-004 상인 납품

```text
지정 무기 의뢰 → 조건 비교 → 즉시 납품 → 강화·수식어 가산금
```

고객 방문 결과형 판매와 역할을 겹치지 않게 한다.

## Graybox

- 대장간·상인·고객 화면
- 실제 골드·재료 상점
- 저장·복귀·방치 보상
- 검투사·군대·모험가 최소 결과
- 자동 단조 저장 호환성

## First Playable

- 20~30분 초기 성장
- 기본 무기·재료 해금
- 세션 목표에 맞는 현실적 강화 구간
- 숨은 무기 조합 1종
- 실패·파괴·복구·판매 경제
- 첫 고객 반복 동기

## Vertical Slice

- 출시 목표 대장간 UI
- 대표 제작→강화→판매 구간
- 검투사 경기 1종
- 접근성·성능 예산과 실제 기기 증거
- Google Play 내부 테스트 AAB
