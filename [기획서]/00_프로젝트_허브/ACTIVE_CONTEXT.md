# Active Context

## 현재 목표

Blacksmith의 프로젝트 운영체계와 모바일 기술 기준을 설치하고, 다음 구현자가 핵심 수직 프로토타입을 바로 계획할 수 있도록 책임 원본과 데이터 경계를 확정한다.

## 현재 상태

- 제품 단계: Concept
- 작업 게이트: Planning·Approval
- 구현 상태: 초기 프로젝트 설정과 데이터 골격
- 검증 상태: JSON 정적 검증 예정, Godot 실행·Android 기기 검증 미실행

## 이번 범위

- Godot 4.7.1 / GDScript
- Android 세로형 모바일
- Google Play / AAB 배포 목표
- 통합 게임 기획서
- 제작·강화·재료·판매 데이터 골격
- 데이터 검증 도구
- 프로젝트 작업 규칙과 개발 게이트

## 보호 결정

- 광클은 피로·과열 불이익이 아니라 피버 보상으로 이어진다.
- 정밀작업은 강제하지 않고 ON/OFF 가능하다.
- 수식어는 기본 제작 시 붙지 않고 강화 이정표에서 발생한다.
- 직원과 무기 수리는 초기 범위에서 제외한다.
- 판매는 고객 직접 방문과 상인 납품 의뢰 두 방식이다.
- 모바일 단순 조작과 빠른 세션을 우선한다.
