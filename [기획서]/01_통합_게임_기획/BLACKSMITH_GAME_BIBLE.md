# Blacksmith 통합 게임 기획서

## 1. 프로젝트 약속

> 한 명의 대장장이가 제한된 하루 작업량으로 장비 한 점을 직접 만들고, 강화 위험과 `+10` 수식어 선택으로 운명을 정한 뒤, 그 장비가 다른 이의 손에서 쌓은 역사를 명성과 다음 의뢰로 돌려받는 모바일 제작 게임.

차별화 문장:

> 장비를 대량 생산하는 게임이 아니라, 장비의 출생·성장·소유·사건 기록을 제작하는 대장장이 게임.

상위 책임 원본은 `docs/superpowers/specs/2026-07-23-project-core-design.md`다. 이 문서는 통합 시스템과 구현 상태를 설명하며 코어 불변 경계를 변경하지 않는다.

## 2. 상태 구분

### 현재 구현

- 철검 제작, 자동 작업, 광클 피버, 제작 정밀 마감
- 일반 강화와 `+10` 단위 특수 강화
- 현재 제품 목표 +100과 수식어 성장
- +11 단계 하락, +30 파괴, 실패 보정
- 균형·안정·폭주 단조
- 골드·재료 거래, 보관함, 자동 단조

현재 제작 기준선:

- 보통·좋음·완벽 기본 공격력: 20·21·22
- 피버 적용 공격력: 21·22·23
- 피버 결과 공격력: ×1.05
- 피버 결과 제작 가치: ×1.03
- 피버 발동 횟수는 기록하지만 결과 보너스는 한 번만 적용되어 중첩되지 않는다.

### 확정 설계·미구현

- 영구 완성도 5등급
- 피로도와 수동 날짜 진행
- 잔여 피로도 50% 이월
- 검투사 의뢰와 +5/+10 납품 판단
- 지연 경기 결과와 설명 가능한 기여 조건
- 영구 세계 장비 기록과 같은 고객 재방문

현재 MVP-003 제품 구현 상태는 `IMPLEMENTATION_NOT_STARTED`다.

## 3. 핵심 설계 원칙

- 한 명의 대장장이만 운용한다.
- 조작은 빠르고 즉각적이며 결과는 점점 묵직해진다.
- 일반 강화는 버튼 입력당 결과 한 번이다.
- `+10` 이정표는 재료·정밀 판정·수식어 선택을 포함한다.
- 영구 완성도는 제작 완료 시 한 번 결정하고 재추첨하지 않는다.
- 낮은 완성도도 강화·수식어·세계 사건 접근을 막지 않는다.
- 판매·납품한 장비는 소유자와 사건 이력을 유지한다.
- 결과 보고는 효과 선택·부족 조건·사건·이력을 분리한다.
- 피로도는 하루 작업량을 제한하지만 성공률·완성도를 불리하게 만들지 않는다.
- 날짜는 플레이어가 직접 하루를 마칠 때만 진행한다.
- 작은 세로 화면에서 한 번에 중요한 판단 하나만 요구한다.

## 4. 핵심 루프

```text
의뢰·세계 소식 확인
→ 장비 제작
→ 영구 완성도 결정
→ 강화와 멈춤 판단
→ +10 수식어 선택
→ 보관·납품
→ 날짜 진행
→ 세계 결과·장비 이력
→ 명성·관계·다음 의뢰
```

장기 확장 루프:

```text
철검 생애 PoC
→ +30 위험 강화 PoC
→ 흉갑·반지 공통 문법 검증
→ 제한된 세계 활동 장비
→ 토너먼트·전쟁 일정
→ 대표작·선택형 복원
```

## 5. 장비 구조

### 상위 계열

- 무기
- 방어구
- 악세서리

초기 장기 대표 장비는 검·흉갑·반지다. 첫 PoC는 철검만 사용한다.

### 장비 정체성

- 계열·세부 종류
- 주재료
- 영구 완성도
- 강화 단계
- 최대 3개의 일반 수식어
- 전문화 방향
- 제작·강화 이력
- 소유자·세계 사건 이력

### 세 성장 축

- 영구 완성도: 어떻게 태어났는가
- 강화 단계: 어떤 위험을 견뎠는가
- 수식어: 어떤 역할과 성질을 얻었는가

## 6. 제작

### 직접 제작

- 터치와 자동 작업으로 진행도를 채운다.
- 빠른 연속 터치는 피버를 채우며 불이익을 주지 않는다.
- 피버는 작업 속도와 현재 Prototype의 제작 결과 보너스에 영향을 준다.
- 제작 시작은 일일 피로도를 소비한다. 터치마다 추가 피로도를 소비하지 않는다.

### 정밀 결과와 영구 완성도

- 정밀 결과: `STANDARD / GOOD / PERFECT`, 자동 제작은 `AUTO`
- 영구 완성도: 5등급, 장비 출생 접두어
- 정밀 결과는 완성도 분포를 바꾼다.
- 정확한 명칭·분포·배율은 PoC 임시 기준값이다.
- 정밀 결과와 영구 완성도를 UI와 데이터에서 분리한다.

기존 `quality_*`는 구현 호환을 위해 유지하되 신규 기록의 완성도 별칭으로 관리하고 legacy 기록은 별도 변환한다.

## 7. 강화

### 일반 강화

- `+10` 단위가 아닌 단계
- 버튼 한 번으로 즉시 1회 판정
- 보조재료·촉매·정밀 판정 미사용

### 특수 강화

- `+10`, `+20`, ... 이정표
- 보조재료와 촉매 선택
- 정밀 판정
- 수식어 추가·성장
- 일반 강화보다 묵직한 연출과 높은 피로도

### 실패·고위험

- +1~+10 실패 시 기본 단계 유지
- +11부터 단계 하락
- +30부터 파괴
- 실패 보정 누적, 성공 시 초기화
- 균형·안정·폭주 단조

실패·보정·위험 수치의 단일 책임 원본은 `data/crafting/enhancement_balance.json`이며, `data/crafting/enhancement_milestones.json`은 +10 특수 강화와 수식어 이정표만 책임진다.

정확한 `+100`은 현재 구현·제품 목표다. 프로젝트 코어는 `+10` 이정표, 점증 위험, 멈춤 판단을 보호하며 최종 상한은 후속 플레이 증거로 재검토할 수 있다.

## 8. 수식어와 재료

- 주재료: 장비 종류와 기본 성능
- 보조재료: 특수 강화 수식어 계열
- 촉매: 성공·성장·가치·위험 조정

초기 계열:

- 숫돌 → 날카로운
- 화염석 → 불타는
- 정령의 심장 → 정령이 깃든

첫 장비 생애 PoC는 날카로움·화염 계열만 사용한다.

## 9. 피로도와 날짜

PoC 기준:

| 작업 | 피로도 |
|---|---:|
| 철검 제작 시작 | 3 |
| 일반 강화 1회 | 1 |
| +10 특수 강화 | 3 |
| 중요 장비 복원 | 5, PoC 미사용 |

- 기본 피로도 20
- 다음 날: `20 + floor(남은 피로도 × 0.5)`
- 별도 이월 상한 없음
- 작업 예약 없음
- 자동 날짜 진행 없음
- 판매·보관·기록 확인은 피로도를 소비하지 않음

피로도는 생존 관리가 아니라 어떤 작품과 의뢰에 오늘의 작업량을 투자할지 결정하는 일정 자원이다.

## 10. 고객·납품·세계 결과

### 고객 직접 방문

- 고객은 목적·필수 조건·추가 목표·기한을 제시한다.
- 기본 대금은 납품 시 지급한다.
- 결과 인센티브는 최소 다음 날의 세계 활동 뒤 지급한다.
- 같은 고객이 장비 결과를 언급하며 재방문한다.

### 첫 검투사 PoC

- +5: 납품 가능
- +10: 첫 수식어와 압도적 결과 가능성
- 결과: DEFEAT / WIN / DECISIVE_WIN
- 난수는 밴드를 바꾸지 않고 같은 밴드의 세부 강도만 바꾼다.

### 원자적 납품

보관함 제거, 소유권, 대금, 즉시 명성, 세계 기록 생성이 모두 성공하거나 모두 이전 상태로 돌아간다. 같은 거래 재시도는 중복 지급을 만들지 않는다.

## 11. 세계 장비 기록

모든 납품 장비는 다음을 기록한다.

- 장비 출생·강화·수식어
- 현재 소유자
- 생애 상태
- 예정·완료 사건
- 결과 보고 상태
- 장비 이력
- 제작자 명성·관계 환류

모든 기록이 계속 사건을 생성하지는 않는다. 제한된 장비만 `ACTIVE_OWNER / EVENT_ELIGIBLE`이며 나머지는 `DORMANT` 상태로 이력을 보존한다.

중요 장비가 파손·분실됐을 때의 선택형 복원은 후속 범위다. 일상적 수리 관리 시스템은 사용하지 않는다.

## 12. 대표작

- 시스템 대표작: 세계가 인정한 역사 장비
- 개인 대표작: 플레이어가 제한적으로 지정한 애착 장비
- 개인 지정은 능력치·판매·강화 보너스를 제공하지 않는다.
- 표시·이력·이벤트 가시성·선택형 복원 우선순위에만 영향을 준다.

첫 PoC에서는 구현하지 않는다.

## 13. 자동 단조

현재 Prototype의 반복 입력 편의 기능이다.

- 목표 강화 단계·반복·단조 방식·재료 설정
- 골드·재료 공유 거래
- 목표 도달 자동 보관
- 부족·가득 참·중지·파괴 처리

코어 보호를 위해 후속 통합 시 `+10`, 위험 개방 구간, 가치 기준에서 강제 정지해야 한다. 첫 장비 생애 PoC에는 참여하지 않는다.

## 14. 모바일 UX·접근성

- Android 세로 720×1280 기준
- 한 화면의 주 행동 1개
- 48dp 상당 터치 영역
- 색상만으로 성공·위험·선택을 전달하지 않음
- 결과 창 자동 닫힘 금지
- 진동·흔들림·모션 감소
- 반복 탭 대안으로 자동 작업 유지
- 정밀 입력 대안으로 느린 게이지·넓은 판정·정밀 보조 제공
- 정밀 보조는 GOOD 분포를 사용하고 PERFECT 희귀 보너스를 자동 제공하지 않음

실제 접근성·Android 통과는 사람과 기기 증거 뒤에만 판정한다.

## 17. 검증 원칙

- 문서와 구현 상태를 분리한다.
- 정상·실패·최소·최대·중복·재시도 반례를 자동 검증한다.
- 자원과 납품 거래는 실패 주입으로 원자성을 검증한다.
- 결과 밴드 세 종류가 모두 도달 가능해야 한다.
- legacy 기록과 자동 단조 template 호환을 회귀 검증한다.
- Godot import, Scene smoke, 모델·통합 테스트를 확인한다.
- Android·접근성·성능·외부 플레이는 미실행을 PASS로 표기하지 않는다.

- Android 실제 기기
- 저장·복귀
- 접근성 장벽 검수
- 성능 예산과 실측
- 콘텐츠 제작 파이프라인
- SWOT·VRIO 재평가
