# Blacksmith 통합 게임 기획서

## 1. 프로젝트 약속

플레이어는 모바일 화면을 빠르게 두드려 무기를 벼리고, 강화 이정표에서 수식어와 숨은 무기를 발견한 뒤, 상황에 맞는 고객 또는 상인에게 판매해 대장간을 성장시킨다.

## 2. 핵심 설계 원칙

- 짧고 즉각적인 터치 반응
- 과도한 광클을 피버로 보상
- 정밀작업은 선택 사항
- 제작보다 강화에서 무기의 개성이 생김
- 복잡한 인력 관리 없이 대장간·상인·고객에 집중
- 좋은 무기를 어디에 팔지 고민하게 함
- 작은 모바일 화면에서 한 번에 하나의 중요한 판단만 요구

## 3. 핵심 게임 루프

`재료 획득 → 기본 무기 제작 → 강화와 수식어 성장 → 고객/상인 판매 → 골드·명성·고급 재료 → 대장간 성장`

## 4. 제작

### 4.1 일반 작업

- 무기와 모루를 터치할 때마다 제작 진행도가 오른다.
- 빠른 연속 터치가 피버 게이지를 채운다.
- 방치 상태에서도 기본 속도로 작업이 진행된다.
- 직원 시스템은 사용하지 않는다.

### 4.2 피버타임

피버타임 동안:
- 터치당 작업량 증가
- 자동 작업 속도 증가
- 마무리 보너스 증가
- 강화 성공률 증가

광클에는 과열·피로 불이익을 두지 않는다.

### 4.3 정밀작업

- 사용 시점: 제작 마무리와 강화
- 기본값: 사용자가 ON/OFF 가능
- ON: 타이밍 게임으로 추가 성공률 또는 품질 보너스
- OFF: 현재 자동 확률로 즉시 판정
- 실패해도 정밀작업을 하지 않은 것보다 큰 손해를 강제하지 않는다.

## 5. 무기 기본 제작

- 무기 종류와 주재료를 선택한다.
- 기본 제작 결과에는 수식어가 없다.
- 이름과 기본 성능은 주재료에서 파생된다.

예:
- 철 + 검 → 철검
- 은 + 검 → 은검
- 운석철 + 검 → 운석검

## 6. 강화와 수식어

초기 강화 이정표:
- +5: 첫 수식어 추가
- +10: 첫 수식어 강화
- +15: 두 번째 수식어 슬롯과 상위 티어 후보
- +20 이후: 프로토타입 검증 후 확정

수식어 후보는 강화 구간에 사용한 보조재료 성질의 누적을 기준으로 결정한다.

## 7. 재료 슬롯

### 주재료
기본 무기 또는 진화 대상 무기를 결정한다.

### 보조재료
수식어 계열과 숨은 조합 조건을 결정한다.

### 촉매
강화 확률, 정밀작업 허용 범위, 희귀 결과 확률을 조절한다.

## 8. 숨은 무기

특정 주재료·보조재료·촉매 조합은 일반 결과 대신 고유 무기를 만든다.

예:
`화염 계열 검 + 정령의 심장 + 살라맨더의 핵 → 살라맨더 화염검`

- 발견 전에는 힌트만 제공한다.
- 최초 발견 후 도감에 레시피가 등록된다.
- 틀린 조합은 가능한 경우 일반 강화 결과로 처리한다.

## 9. 판매

### 9.1 고객 직접 방문

- 고객은 상황과 목적을 요청한다.
- 기본 대금은 고정이다.
- 플레이어가 보유 무기 중 하나를 선택한다.
- 무기 적합성과 활동 결과에 따라 나중에 인센티브를 받는다.

### 9.2 상인 납품 의뢰

- 상인이 원하는 무기 종류를 지정한다.
- 기본 가격에 강화 등급, 수식어, 희귀도 보너스가 붙는다.
- 납품 즉시 정산한다.
- 상인 메뉴에서는 재료·촉매·힌트 아이템도 구매할 수 있다.

## 10. 고객

### 검투사
- 주요 보상: 명성, 인기, 경기 인센티브
- 플레이어 무기로 승리하면 단골 확률이 오른다.
- 경기를 관람하고 게임 내 재화로 결과 베팅이 가능하다.

### 군대
- 주요 보상: 안정적인 골드
- 납품 장비가 방어 전투력에 기여한다.
- 방어 라이프가 0이 되면 현재 전쟁 시즌이 종료된다.
- 영구 대장간 성장은 보존한다.

### 모험가
- 주요 보상: 고급·희귀 재료
- 지역 요구에 맞는 무기를 주면 성공률과 보상이 오른다.

## 11. 대장간과 상인

대장간:
- 제작, 강화, 인벤토리, 설비 성장

상인:
- 고객 요청 관리
- 납품 의뢰
- 재료·촉매 구매
- 숨은 조합 힌트 구매

## 12. 모바일 UX

- 기본 세로 화면 720×1280
- 핵심 제작 대상은 엄지로 누르기 쉬운 화면 중앙·하단에 배치
- 한 손 플레이를 기본으로 고려
- 주요 터치 대상은 충분한 크기와 간격 확보
- 노치와 시스템 제스처 안전 영역 대응
- 정밀작업은 짧고 명확한 판정 구간 제공
- 진동·화면 흔들림은 끌 수 있어야 한다.
- 장시간 연속 탭을 강제하지 않고 방치 진행을 병행한다.

## 13. Android·Google Play

- Godot 4.7.1 stable / GDScript
- Android App Bundle 배포
- OpenJDK 17 및 Android SDK 기반 export
- 2026년 8월 31일 이후 신규 앱 제출을 고려해 API 36 이상을 목표로 준비
- 서명키는 저장소에 커밋하지 않는다.
- 실제 배포 전 Google Play 최신 정책을 다시 확인한다.

## 14. Prototype 수직 범위

### MVP-001 제작 터치

`철 고정 → 철검 광클·자동 제작 → 피버 → 선택적 정밀 마감 → 철검 완성 → 다시 제작`

- 현재 상태: 구현, 자동·시각·실기기 검증 중
- 책임 경로: `scenes/main/main.tscn`, `scripts/forging/`, `scripts/ui/`, `data/crafting/forging_balance.json`
- 범위 문서: `docs/MVP-001_SCOPE.md`

### MVP-002 강화 이정표

`철검 → 강화 재료 선택 → 피버·정밀 강화 → +5 → 예리함 계열 첫 수식어`

### MVP-003 첫 판매

`완성 무기 선택 → 방문 검투사에게 지급 → 고정 기본금 → 경기 결과 → 인센티브·명성`

세 범위를 분리해 제작 손맛을 먼저 검증하고 강화·판매 복잡도가 핵심 터치 경험을 가리지 않게 한다.

## 16. 검증 기준

### MVP-001
- 플레이어가 광클과 피버의 관계를 설명할 수 있다.
- 정밀작업을 끄고도 철검을 완성할 수 있다.
- 정밀작업을 켜면 타이밍에 따라 마감 결과가 달라진다.
- 실제 Android 기기에서 세로 화면과 터치 입력이 안정적으로 동작한다.

### Prototype 전체
- 재료 선택이 수식어 후보에 영향을 준다는 것을 확인할 수 있다.
- 고객 방문과 상인 납품의 차이를 한 번의 플레이로 이해한다.
