# Blacksmith 통합 게임 기획서

## 1. 프로젝트 약속

플레이어는 모바일 화면을 빠르게 두드려 무기를 벼리고, 고위험 강화와 특수 재료 선택으로 가치가 급상승하는 무기를 만든 뒤 상황에 맞는 고객 또는 상인에게 판매해 대장간을 성장시킨다.

## 2. 핵심 설계 원칙

- 짧고 즉각적인 터치 반응
- 과도한 광클을 피버로 보상
- 제작 마감과 특수 강화의 정밀 판정은 짧고 명확하게 제공
- 기본 무기는 단순하고, 강화에서 무기의 개성과 가치가 생김
- 고단계 성공은 효과·가격 상승이 크게 느껴져야 함
- 고위험은 하락·파괴·비용·보호 선택으로 명확히 표시
- 자동 단조는 반복 피로를 줄이되 수동 선택과 광클의 의미를 지우지 않음
- 복잡한 인력 관리 없이 대장간·상인·고객에 집중
- 작은 모바일 화면에서 한 번에 하나의 중요한 판단만 요구

## 3. 핵심 게임 루프

`재료·골드 획득 → 기본 무기 제작 → 강화·수식어 성장 → 보관 → 고객/상인 판매 → 골드·명성·고급 재료 → 대장간 성장`

현재 Prototype은 `제작 → +100 강화 → 보관 → 자동 단조`까지 구현하고 판매 루프 연결을 다음 단계로 둔다.

## 4. 제작

### 4.1 일반 작업

- 무기와 모루를 터치할 때마다 제작 진행도가 오른다.
- 빠른 연속 터치가 피버 게이지를 채운다.
- 방치 상태에서도 기본 속도로 작업이 진행된다.
- 직원 시스템은 사용하지 않는다.

### 4.2 피버타임

피버타임 동안:

- 터치당 작업량 증가
- 자동 작업 속도 증가
- 마무리 보너스 증가

광클에는 과열·피로 불이익을 두지 않는다. 강화 피버 보정은 실제 구현 범위가 생길 때 별도 데이터 계약으로 추가한다.

### 4.3 정밀 마감

- 제작 마무리에서 ON/OFF 가능
- ON: 타이밍에 따라 완벽·좋음·보통 결과
- OFF: 기본 결과로 즉시 마감
- 정밀 마감 실패가 비사용보다 과도한 손해를 강제하지 않음

## 5. 기본 무기 제작

- 무기 종류와 주재료를 선택한다.
- 기본 제작 결과에는 수식어가 없다.
- 이름과 기본 성능은 주재료에서 파생된다.

예:

- 철 + 검 → 철검
- 은 + 검 → 은검
- 운석철 + 검 → 운석검

Prototype 구현은 철검을 우선한다.

## 6. 강화 구조

### 6.1 일반 강화

- 대상: +10 단위가 아닌 모든 단계
- 조작: 버튼 한 번으로 즉시 판정
- 보조재료·촉매·정밀 판정 미사용
- 이전 특수 강화의 선택 효과가 일반 강화에 누출되지 않음

### 6.2 특수 강화

- 대상: +10·+20·…·+100
- 보조재료·촉매 선택
- 정밀 판정
- 수식어 추가 또는 티어 성장
- 일반 강화와 다른 화면·설명·위험 강조

정밀 강화는 특수 강화에서만 적용한다. 현재 Prototype에서 OFF 즉시 판정은 제공하지 않으며 특수 강화의 핵심 조작으로 유지한다.

### 6.3 점진 성장과 가치

- 공격력 증가량은 고정 수치가 아니라 현재 강화 적용 공격력과 목표 구간을 기준으로 계산한다.
- 일반 강화보다 특수 강화의 성장률이 크다.
- 고단계일수록 공격력 증가량, 판매가, 강화 비용이 가속된다.
- 화면에서 현재 값과 다음 성공 시 공격력·판매가·비용·확률을 비교한다.
- 실제 계산값은 `data/crafting/enhancement_balance.json`이 책임진다.

## 7. 강화 실패와 고위험

- +1~+10은 학습 구간으로 실패 시 기본적으로 단계 유지
- +11부터 단계 하락 가능
- +30부터 무기 파괴 가능
- 고단계일수록 하락·파괴 위험과 하락 단계가 증가
- 실패 시 다음 성공률 보정 누적
- 성공 시 실패 보정 초기화
- 파괴된 무기는 보관·판매할 수 없고 강화 세션 종료
- 무기 수리 시스템은 사용하지 않는다.

확률표·보정·하락 단계·비용은 JSON 데이터가 책임진다.

## 8. 단조 방식

### 균형 단조

추가 보정 없이 기본 성공률·비용·성장·위험을 사용한다.

### 안정 단조

비용이 증가하지만 파괴를 막고 단계 하락 위험을 줄인다. 안전 선택은 무료 기본값이 아니라 명확한 비용 교환이다.

### 폭주 단조

- 성공 시 낮은 확률로 총 2단계 상승
- 도약이 없으면 1단계 상승
- 특수 강화에서 사용 불가
- +9·+19·+29·…처럼 2단계 도약이 +10 단위 이정표를 건너뛸 구간에서 사용 불가
- 공격력·판매가·파괴 위험을 직접 배율로 올리는 이전 폭주 구조는 사용하지 않는다.

## 9. 재료 슬롯

### 주재료

기본 무기와 기본 성능을 결정한다.

### 보조재료

특수 강화의 수식어 계열을 결정한다.

- 숫돌 → 날카로운 계열
- 화염석 → 불타는 계열
- 정령의 심장 → 정령이 깃든 계열

### 촉매

특수 강화의 성공률·성장·가치·하락·파괴를 조정한다.

- 촉매 없음
- 살라맨더의 핵
- 수호 가루
- 폭주의 불씨

각 촉매의 가격과 실제 효과는 `data/crafting/materials.json`이 책임진다. 자동 단조에서 지정 촉매 재고가 없으면 촉매 없이 계속한다.

## 10. 수식어 성장

- +10: 첫 수식어 1티어
- +20: 첫 수식어 2티어
- +30: 두 번째 수식어 1티어
- +40: 두 번째 수식어 2티어
- +50: 세 번째 수식어 1티어
- +60: 세 번째 수식어 2티어
- +70: 첫 수식어 3티어
- +80: 두 번째 수식어 3티어
- +90: 세 번째 수식어 3티어
- +100: 모든 수식어 4티어

새 수식어를 추가할 때 이미 보유한 수식어와 다른 후보를 우선한다. 실제 티어 효과는 `data/crafting/affixes.json`이 책임진다.

## 11. 무기 보관함

- 최대 6개
- 강화 도중 원하는 단계에서 수동 보관 가능
- 자동 단조 목표 도달 시 자동 보관
- 이름·강화 단계·기본/강화/최종 공격력·판매가·누적 비용·예상 손익·수식어·촉매·마감 품질 표시
- 파괴 무기 보관 불가
- 보관함은 판매 루프의 선택 입력이 된다.

## 12. 자동 단조

### 설정

- 목표 강화 단계
- 보관함이 찰 때까지 반복 여부
- 단조 방식
- 특수 강화 보조재료
- 특수 강화 촉매

### 실행

1. 현재 무기를 목표 단계까지 빠르게 강화한다.
2. 특수 강화에서 지정 재료 재고를 확인한다.
3. 재고가 없으면 해당 슬롯을 비운 상태로 계속한다.
4. 목표 도달 시 보관한다.
5. 반복 설정이면 새 철검으로 다시 시작한다.
6. 골드 부족·보관함 가득 참·수동 중지에서 종료한다.
7. 파괴 시 반복 설정이면 새 철검으로 재시작하고 아니면 종료한다.

자동화는 수동 플레이를 완전히 대체하는 무료 최적해가 아니어야 한다. 경제·해금·속도·목표 선택은 후속 플레이테스트로 조정한다.

## 13. 숨은 무기

특정 주재료·보조재료·촉매 조합은 일반 결과 대신 고유 무기를 만든다.

예:

`화염 계열 검 + 정령의 심장 + 살라맨더의 핵 → 살라맨더 화염검`

- 발견 전에는 힌트만 제공
- 최초 발견 후 도감에 레시피 등록
- 틀린 조합은 가능한 경우 일반 결과 처리
- 실제 변환은 First Playable 전 별도 범위에서 검증

## 14. 판매

### 고객 직접 방문

- 고객은 상황과 목적을 요청한다.
- 기본 대금은 고정이다.
- 플레이어가 보관 무기 중 하나를 선택한다.
- 무기 적합성과 활동 결과에 따라 인센티브를 받는다.

### 상인 납품 의뢰

- 상인이 원하는 무기 종류·강화·수식어 조건을 지정한다.
- 기본 가격에 강화 단계·수식어·희귀도 보너스가 붙는다.
- 납품 즉시 정산한다.
- 재료·촉매·힌트 아이템 구매와 연결한다.

두 판매 방식은 결과형 장기 보상과 조건형 즉시 정산으로 역할을 분리한다.

## 15. 고객

### 검투사

- 주요 보상: 명성·인기·경기 인센티브
- 플레이어 무기로 승리하면 단골 확률 증가
- Prototype 다음 구현 대상
- 후속 확장 후보: 경기 관람과 게임 내 재화 베팅

경기 관람·베팅 기능은 삭제하지 않고 보존한다. 다만 MVP-003 기본 판매 범위에는 포함하지 않으며, 판매 루프가 검증된 뒤 별도 PoC·경제 가드레일·사용자 승인으로 구현 여부와 범위를 확정한다.

### 군대

- 주요 보상: 안정적인 골드
- 납품 장비가 방어 전투력에 기여
- 방어 라이프가 0이면 현재 전쟁 시즌 종료
- 영구 대장간 성장은 보존

### 모험가

- 주요 보상: 고급·희귀 재료
- 지역 요구에 맞는 무기는 성공률·보상 증가

## 17. 모바일 UX·접근성

- 기본 세로 화면 720×1280
- 핵심 제작 대상은 엄지 접근이 쉬운 중앙·하단 배치
- 주요 터치 대상의 충분한 크기와 간격
- 노치·시스템 제스처 안전 영역 대응
- 특수 강화는 재료 선택→효과·가격·위험 확인→정밀 판정 순서
- 긴 화면은 세로 스크롤 허용
- 진동·화면 흔들림은 끌 수 있어야 함
- 색상만으로 성공·위험·선택 상태를 구분하지 않음
- 자동 단조 중지와 현재 상태를 항상 확인 가능
- 장시간 연속 탭을 강제하지 않고 자동 작업을 병행

실제 접근성 통과는 Android 기기와 사람 검수 증거가 있을 때만 판정한다.

## 19. Prototype 범위

### MVP-001 제작

`철 고정 → 광클·자동 제작 → 피버 → 선택적 정밀 마감 → 철검 완성`

- 구현·자동 검증 PASS 이력
- 시각·Android 미검증
- 범위: `docs/MVP-001_SCOPE.md`

### MVP-002 +100 강화·보관·자동 단조

`철검 → 일반/특수 강화 → 위험·수식어 성장 → 보관 → 목표 자동 반복`

- 구현·자동 검증 PASS 이력
- 장기 밸런스·성능·Android 미검증
- 범위: `docs/MVP-002_SCOPE.md`

### MVP-003 방문 검투사 판매

`보관 무기 선택 → 요청 → 고정 기본금 → 경기 결과 → 인센티브·명성·단골 확률`

- 다음 구현
- 경기 관람·베팅은 기본 범위가 아니라 후속 확장 후보로 보존

### MVP-004 상인 납품

`지정 조건 → 제작·강화 → 조건 비교 → 즉시 납품 → 가산금`

- MVP-003 이후

## 20. 현재 제외와 보류

### 현재 프로젝트 제외

- 직원
- 무기 수리
- 멀티플레이·서버
- 결제·광고
- iOS
- 복잡한 장비 부위
- 제작 중 영구 파괴
- 승인되지 않은 신규 시스템

강화 중 파괴는 +30 이상 고위험 결과로 현재 구현에 포함한다.

### 기능 보존·후속 검토

- 검투사 경기 관람·게임 내 재화 베팅: 기능 아이디어를 보존하되 MVP-003 기본 범위에서는 제외한다. 판매 루프·경제·연령 등급 위험을 먼저 검증하고 별도 승인 뒤 구현한다.
- 숨은 무기 실제 변환: First Playable 전 별도 범위에서 검증한다.
- 강화 피버 보정: 실제 구현 범위와 데이터 계약이 승인될 때 재개한다.

## 21. 검증 기준

### 제작

- 광클과 피버 관계를 설명할 수 있다.
- 정밀 마감을 끄고도 철검을 완성할 수 있다.
- 정밀 마감을 켜면 결과가 달라진다.
- Godot 파싱과 제작 모델 테스트가 통과한다.
- 실제 Android 세로 화면·터치가 안정적으로 동작한다.

### 강화·보관·자동 단조

- 일반 강화와 특수 강화 경계를 사용자가 이해한다.
- 일반 강화에 재료·촉매·정밀 효과가 누출되지 않는다.
- 다음 효과·가격·비용·위험을 확인할 수 있다.
- +11 하락·+30 파괴·실패 보정·안정 보호가 실제 확률 계약과 일치한다.
- 폭주 도약이 특수 강화 이정표를 건너뛰지 않는다.
- +100 수식어 성장 결과가 데이터와 일치한다.
- 자동 단조가 재료 부족 fallback, 목표 보관, 반복, 중지, 파괴를 유한하게 처리한다.
- Godot 파싱·Scene·모델·JSON 검증이 통과한다.
- 장시간 자동 반복에서 무한 루프·프레임 정지·메모리 증가가 없다.
- 실제 Android에서 재료 선택·스크롤·정밀 입력·자동 중지가 안정적으로 동작한다.

### 판매 진입

- 보관함 무기가 고객·상인 선택 입력으로 연결된다.
- 방문 고객과 납품 의뢰의 보상·시간·판단 역할이 겹치지 않는다.
- 판매가 강화 가치와 재료 경제를 다시 공급한다.
- 경기 관람·베팅 확장 여부는 MVP-003 기본 판매 루프 검증 뒤 별도 PoC로 판정한다.

자동 검증, 실제 렌더, Android, 접근성, 성능, 외부 플레이테스트 상태를 분리해 기록한다.
